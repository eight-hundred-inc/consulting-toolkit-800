# SLIDE-PATTERN-sidebar-toc-panel

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスターと組み合わせてAIツールに渡すことで、このパターンのスライドを生成できます。タイトルエリア・ページ番号・装飾はスライドマスター（branded-pptx / html-artifactテーマ）で定義されるため、このファイルには含みません。プレビュー .html は生成当時の見本であり非規範。見た目は選択マスターが決めます。

## Overview

**パターン名：** sidebar-toc-panel
**概要：** スライド全体を左右に二分割し、左側に「INDEX」などのセクション見出しを縦書きまたは横書きで大きく置き、右側に番号付き目次項目を縦に並べる目次専用レイアウト。
**適したシーン：** プレゼンテーションの目次（INDEX）スライド、セクション切り替えのアジェンダ表示、資料の章立て紹介。

## Structure（構造）

コンテンツエリアを左右に分割する。左パネルは強調背景の上にセクション見出し（例：「INDEX」）を中央揃えで大きく表示し、右パネルには番号バッジ＋項目テキストの組み合わせを縦方向に等間隔で並べる。

```yaml
layout: two-column
zones:
  - role: left_panel
    width: 28%
    content: セクションラベル（例：「INDEX」「目次」、縦中央揃え、強調背景）
  - role: right_panel
    width: 72%
    content: 目次項目（番号バッジ＋項目テキスト）×4〜7
reading_order: 左パネル → 右パネル（上から下へ項目を列挙）
notes: 目次項目は4〜7項目を推奨。それ以上になる場合は文字サイズを縮小するか2列化を検討する
```

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| 左パネル背景 | 左側全域 | 強調色の背景でセクション感を視覚的に示す |
| セクションラベル | 左パネル中央 | 「INDEX」「目次」などのラベルを縦中央に大きく表示する |
| 右パネル | 右側全域 | 白またはニュートラル背景で目次項目を並べる |
| 番号バッジ | 各目次項目の左 | 章番号を正方形バッジで表示し、視覚的アクセントとなる |
| 目次テキスト | 番号バッジの右 | 章・セクションの題名を表示する（フォントは大きめ） |

## Usage Guide（AIへの使い方）

このパターンをAIに指示する際のプロンプト例：

> 「SLIDE-PATTERN-sidebar-toc-panelのレイアウトで、プレゼンテーションの目次スライドを作成してください。左パネルに「INDEX」を大きく表示し、右パネルに5つの章タイトルを番号付きで並べてください。デザインはスライドマスターに従ってください。」

**注意点：**
- 目次項目数は4〜7項目が視認性のうえで最適。それ以上になる場合はフォントサイズを小さくするか、2列化を検討する。
- 番号バッジは強調色の正方形または角丸正方形にし、テキストを白抜きにする。
- 左パネルのセクションラベルは縦中央揃え・大きめのフォントで存在感を持たせる。
- 右パネルの目次項目は等間隔に配置し、上下に余白を設けてゆったりと見せる。
