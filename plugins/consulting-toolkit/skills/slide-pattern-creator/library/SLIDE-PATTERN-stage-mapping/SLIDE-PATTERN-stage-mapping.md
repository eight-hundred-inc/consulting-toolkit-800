# SLIDE-PATTERN-stage-mapping

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスターと組み合わせてAIツールに渡すことで、このパターンのスライドを生成できます。タイトルエリア・ページ番号・装飾はスライドマスター（branded-pptx / html-artifactテーマ）で定義されるため、このファイルには含みません。

## Overview

**パターン名：** stage-mapping
**概要：** 左に3段階の連続ステージ（番号付き）を縦に並べ、各ステージに「概要」と「役割」の説明カラムを追加し、右側に本提案のアプローチとしてまとめボックス（AとBラベル）で対応関係を示す4カラム構成のマッピングレイアウト。
**適したシーン：** 業務効率化→高度化→革新のような段階モデル、既存フレームワークに自社アプローチを重ねる説明、複数のステージを横断してどこにどう手を打つかを示すスライド。

## Structure（構造）

コンテンツエリアを大きく左3/4（分析エリア）と右1/4（提案エリア）に分ける。左3/4はさらに4列（ステージ番号＋段階名／概要／生成AIの役割／中矢印）に分割し、3段のステージを縦に積む。右1/4に括弧付きのブラケットで2グループ（A、B）を作り、対応関係を視覚化する。

    structure:
      layout: left-3col-mapping-right-callout
      left_area:
        width: 75%
        columns:
          - name: "ステージ番号＋段階名"
            width: 25%
          - name: "概要"
            width: 30%
          - name: "生成AIの役割"
            width: 35%
        rows: 3  # ステージの数
      arrow_indicator: 
        position: "各行の右端"
        style: "▶ アイコン"
      right_area:
        width: 25%
        content: "2グループのブラケット（A/B）とラベル＋説明"

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| ステージ番号バッジ | 左端の縦列 | 1・2・3の連番でステージを示す（ブルー円） |
| 段階名 | ステージ番号の右 | ステージのラベル（業務効率化・業務高度化・業務革新 等） |
| 概要 | 中央列 | 各ステージが何を扱うかの1〜2行説明 |
| 役割ラベル | 右列（強調テキスト） | 生成AIの役割をアクセントカラーで強調（時間の圧縮・ヒト×AIの協働 等） |
| 役割サブテキスト | 役割ラベル直下 | 役割の詳細説明（1〜2行） |
| 段階間区切り線 | 各ステージ行の間 | 薄い点線でステージを視覚的に区切る |
| 提案ブラケット | 右端 | 複数ステージを束ねる大きな括弧（}）とA/Bラベル |
| 提案ラベル | ブラケット横 | 本提案の対応アプローチ（例：AIスキル化、グランドデザイン） |

## Usage Guide（AIへの使い方）

このパターンをAIに指示する際のプロンプト例：

> 「SLIDE-PATTERN-stage-mappingのレイアウトで、3段階のステージ（効率化・高度化・革新）を左に配置し、右側に本提案アプローチ2つを対応関係で示してください。デザインはスライドマスターに従ってください。」

**注意点：**
- ステージ数は3が視覚的にちょうどいい。4以上はレイアウトが窮屈になる
- 右側のブラケットは「複数のステージを束ねる」役割なので、必ず2つ以上のステージにまたがるように配置する
- 「概要」列は簡潔に。ここが冗長になると全体が読みにくくなる
