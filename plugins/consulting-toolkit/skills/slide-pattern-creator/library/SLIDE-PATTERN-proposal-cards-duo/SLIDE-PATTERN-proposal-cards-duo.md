# SLIDE-PATTERN-proposal-cards-duo

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスター（表紙・タイトル行・フッター・配色・フォント）はマスター側（branded-pptx / html-artifactテーマ等）で定義されるため、このファイルには含みません。

## Overview
**パターン名：** proposal-cards-duo
**概要：** 施策・仮説カードを2枚横並びにし、各カード内の行ラベル（打ち手／先行事例／効果 等）を左右で揃えて対比させ、下部に両者の関係性を説明するコールアウトを敷く構造。
**適したシーン：** 2つの施策・仮説・選択肢を並列で提示しつつ、それぞれの内容を同じ観点（打ち手・根拠・効果）で比較させたいスライド。単独の施策紹介ではなく「2案を並べて見せる」場面に向く。
**対応する図解指名：** 施策カード2枚対比
**下流実装ヒント：** html-artifact: Proposal Card 2枚 ／ pptx: 罫線カード2枚＋行ラベル

## Structure（構造）
コンテンツエリア上部に導入文を1〜2文置き、その下に2枚のカードを横並びで配置する。各カードはヘッダ帯（番号チップ＋見出し）と、その下に続く複数の行（行ラベル＋行本文のペア）で構成され、2枚のカードは同じ行ラベルの並び順で揃える。最下部には2カード幅いっぱいのコールアウト帯を置き、2つの施策・仮説の関係性（補完関係・優先順位等）を説明する。

    structure:
      layout: two-column-cards
      intro: 導入文（1〜2文）
      cards:
        count: 2
        each:
          header:
            chip: 番号（例：仮説①）
            title: カード見出し（施策・仮説名）
          rows:
            - label: 打ち手
              body: 具体的なアクション内容
            - label: 先行事例
              body: 根拠となる事例・実績
            - label: 効果（利益をどう動かすか 等）
              body: 期待される効果の説明
      bottom_band:
        label: 両者の関係
        body: 2つの施策・仮説がどう補完し合うか、あるいはどちらを優先すべきかの説明

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| 導入文 | コンテンツエリア最上部 | 2枚のカードがどのような文脈で導かれたかを説明する前置き |
| 番号チップ | 各カードヘッダ左 | カードの通し番号（仮説①／仮説② 等）を明示するタグ |
| カード見出し | 各カードヘッダ | 施策・仮説の名称そのもの |
| 行ラベル（打ち手） | 各カード1行目 | その施策・仮説が具体的に何をするアクションかを示す |
| 行ラベル（先行事例） | 各カード2行目 | 施策の実現可能性・説得力を裏付ける根拠・実績 |
| 行ラベル（効果） | 各カード3行目 | 施策を実行した結果どのような変化・利益が生まれるかを示す |
| 関係性コールアウト | カード群の直下・全幅 | 2つの施策・仮説を並べて終わりにせず、両者の関係（補完・優先順位・組み合わせ方）を明言する |

## Usage Guide（AIへの使い方）

> 「SLIDE-PATTERN-proposal-cards-duoのレイアウトで、カード①に［施策・仮説A：打ち手／根拠／効果］、カード②に［施策・仮説B：打ち手／根拠／効果］を入れ、下部のコールアウトに［AとBの関係性・組み合わせ方］を記入してください。デザインはスライドマスターに従ってください。」

**注意点：**
- 2枚のカードの行ラベルは必ず同じ並び順・同じ項目数で揃える。片方だけ行数が多いと対比構造が崩れる。
- 行ラベルは「打ち手／先行事例／効果」の3行が標準だが、内容に応じて「課題／解決策／体制」等に差し替えてよい。ただし2枚とも同じラベル構成にすること。
- 下部の関係性コールアウトは省略可能だが、2案が独立で無関係に見えてしまう場合は必須で入れる。
