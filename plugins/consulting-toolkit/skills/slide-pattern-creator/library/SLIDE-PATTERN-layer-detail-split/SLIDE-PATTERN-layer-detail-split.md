# SLIDE-PATTERN-layer-detail-split

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスター（表紙・タイトル行・フッター・配色・フォント）はマスター側（branded-pptx / html-artifactテーマ等）で定義されるため、このファイルには含みません。プレビュー .html は生成当時の見本であり非規範。見た目は選択マスターが決めます。

## Overview
**パターン名：** layer-detail-split
**概要：** 左1/3に縦積み（または階層）のレイヤー構造図を置き、右2/3に各層と行対応した詳細説明を並べる、構造と解説を左右で対応させるレイアウト。
**適したシーン：** システム構成・成熟度モデル・機能階層など「層に分かれた構造」を示しつつ、各層が何を意味するか・どう機能するかを1層ずつ説明したいスライド。層の並び順そのものに意味がある内容に向く。
**対応する図解指名：** レイヤー構造＋詳細
**下流実装ヒント：** html-artifact: レイヤー積層図＋テキスト ／ pptx: 積層shape＋説明列

## Structure（構造）
コンテンツエリアを左右に分割し、左カラム（約1/3）に複数の層を縦に積んだ構造図を、右カラム（約2/3）に各層と縦位置を揃えた詳細説明の行を配置する。左の層ボックスと右の説明行は必ず同じ高さ・同じ行数で揃え、視線が左右に往復したときに層と説明が1対1で対応するようにする。層は3〜6段を目安とし、下から上（または上から下）へ意味的な順序を持たせる。

```yaml
layout: layer-detail-split
zones:
  - role: layer_stack
    width: "1/3"
    position: left
    content: 層ボックス（縦積み3〜6段。各段が1つの層）
  - role: detail_rows
    width: "2/3"
    position: right
    content: 詳細行（層と縦位置を揃えた説明。層数と同数）
reading_order: 左（層ボックス、積み順）→ 右（対応する詳細行）
notes: 左の各層ボックスと右の各詳細行は縦位置・行数を一致させる
```

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| 層ボックス | 左カラム・縦積み | 構造を構成する各層の名称。積み順そのものが階層・段階の意味を持つ |
| 詳細行 | 右カラム・各層と同じ縦位置 | 対応する層が担う役割・要素・具体例を1層につき1行で説明する |
| 行対応の揃え | 左右カラム間 | 層と説明を1対1で対応させ、どの説明がどの層のものか一目で分かるようにする |

## Usage Guide（AIへの使い方）

> 「SLIDE-PATTERN-layer-detail-splitのレイアウトで、左に［層1／層2／層3／層4］の縦積み構造図、右に各層の詳細説明を行対応で入れてください。層の並び順は［下位→上位 等の意味］に従い、左右の行数を揃えてください。デザインはスライドマスターに従ってください。」

**注意点：**
- 左の層ボックス数と右の詳細行数は必ず一致させる。数がずれると行対応が崩れて読み手が対応関係を追えなくなる。
- 層は3〜6段が適切。7段以上になる場合は層をグルーピングして減らすか、別スライドに分割する。
- 積み順に意味（下位→上位、前段→後段 等）を持たせ、順序が逆でも成立するような並びにはしない。
