# SLIDE-PATTERN-agenda-current-highlight

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスター（表紙・タイトル行・フッター・配色・フォント）はマスター側（branded-pptx / html-artifactテーマ等）で定義されるため、このファイルには含みません。プレビュー .html は生成当時の見本であり非規範。見た目は選択マスターが決めます。

## Overview
**パターン名：** agenda-current-highlight
**概要：** 番号付きの章リストを縦に並べ、現在進行中の章のみを濃色・太字で強調し、他の章はグレーアウトして表示するアジェンダレイアウト。
**適したシーン：** 複数章から成るプレゼンの冒頭で全体構成を示す、または各章の切り替わり時に「今どこにいるか」を示すために章間で繰り返し使用するスライド。
**対応する図解指名：** アジェンダ（現在地強調）
**下流実装ヒント：** html-artifact: TOC ／ pptx: 番号リスト＋現在行のみ濃色塗り・太字、他行はグレー文字

## Structure（構造）
コンテンツエリアは単一カラムの縦リストで構成される。上部に区切りの罫線を1本引き、その下に章番号と章名のペアを縦に等間隔で並べる。リスト中の1項目（現在の章）だけを強調し、他の項目はすべて同じトーンで統一して視線を現在項目に集中させる。章数は3〜6程度を想定する。

    layout: single-column-list
    zones:
      - role: top-rule
        content: 上部の区切り線
      - role: list
        content: 章番号＋章タイトルのリスト項目 ×3〜6（各項目は current | inactive のいずれかの状態）
        emphasis: 現在の章（current）は強調表示（任意で左サイドバーを追加）。他の章（inactive）は非強調表示で統一する
    reading_order: 上部区切り線 → 章リスト（上から下）
    notes: 章が切り替わるたびに state のみを差し替えて同一レイアウトを繰り返し使用する

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| 上部罫線 | コンテンツエリア最上部 | リストの開始位置を示す区切り線 |
| 章番号 | 各リスト項目の左端 | 章の通し番号（発表順序を示す） |
| 章タイトル | 各リスト項目の番号右側 | 章の名称（現在地把握のための主要テキスト） |
| 強調項目（現在の章） | リスト中の1項目のみ | 聴衆に「今この章の話をしている」ことを伝える視覚的アンカー |
| 非強調項目（他の章） | 強調項目以外の全項目 | 全体構成の中での相対的な位置づけを示す背景情報（グレーアウトで後退させる） |

## Usage Guide（AIへの使い方）

> 「SLIDE-PATTERN-agenda-current-highlightのレイアウトで、章リスト［章1〜章Nの名称］を入れ、現在の章は［章番号］にしてください。デザインはスライドマスターに従ってください。」

**注意点：**
- 強調は「濃色＋太字」を基本とし、他項目とのコントラストを最大化する。強調項目を増やす（2項目以上）とスライドの目的が曖昧になるため必ず1項目に限定する。
- 章数が多い（7以上）場合はリストが縦に詰まりすぎるため、間隔を詰めるか章をグルーピングして減らすことを検討する。
- 章間で繰り返し使うことが前提のパターンなので、章名・番号のテキストと state（現在／非現在）以外の構造要素は毎回変えないこと。
