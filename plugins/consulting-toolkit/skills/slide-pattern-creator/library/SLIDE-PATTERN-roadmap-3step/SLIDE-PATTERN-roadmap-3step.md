# SLIDE-PATTERN-roadmap-3step

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスター（表紙・タイトル行・フッター・配色・フォント）はマスター側（branded-pptx / html-artifactテーマ等）で定義されるため、このファイルには含みません。プレビュー .html は生成当時の見本であり非規範。見た目は選択マスターが決めます。

## Overview
**パターン名：** roadmap-3step
**概要：** STEPヘッダー・期間・内容・成果物欄を持つカードを3枚横並びにし、進行順を示すロードマップレイアウト。
**適したシーン：** プロジェクトの進め方や導入ステップを3段階で提示し、各段階の期間・内容・成果物をまとめて見せたいスライド。
**対応する図解指名：** 3ステップロードマップ
**下流実装ヒント：** html-artifact: Roadmap ／ pptx: 3枚の角丸矩形カードを横並び＋間に矢印コネクタ、STEP1のヘッダーのみ濃色塗りで強調

## Structure（構造）
コンテンツエリアは3枚のステップカードを横一列に等幅で並べ、カード間に進行方向を示す矢印を挟む。各カードは縦方向に4段で構成される：ヘッダー（STEP番号ラベル＋期間チップ）、タイトル（そのステップで行うことの見出し）、本文（具体的な内容説明）、成果物欄（そのステップで得られるアウトプットを明示する下部ボックス）。最初のステップ（STEP1）のヘッダーのみ強調表示にし、直近の着手ステップであることを示せる。

```yaml
layout: three-column-cards-with-arrows
zones:
  - role: card
    content: ステップカード ×3（横一列・等幅）。各カードはヘッダー（STEP番号＋期間チップ）／タイトル／本文／成果物欄の縦4段構成
  - role: connector
    content: 進行方向を示す矢印（カード間）
reading_order: STEP1 → 矢印 → STEP2 → 矢印 → STEP3
notes: STEP1のヘッダーのみ強調表示可、他は通常表示とする
```

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| STEPヘッダー | 各カード最上部 | ステップ番号と所要期間を示し、進行順序を明示する |
| 期間チップ | ヘッダー内右側 | そのステップにかかる想定期間を数値で示す |
| ステップタイトル | ヘッダー直下 | そのステップで何を行うかの見出し |
| 本文 | タイトル直下 | ステップの具体的な取り組み内容を説明する |
| 成果物欄 | 各カード最下部 | そのステップの完了時に得られる具体的なアウトプットを明示する |
| 矢印コネクタ | カードとカードの間 | ステップが順番に進行することを視覚的に示す |

## Usage Guide（AIへの使い方）

> 「SLIDE-PATTERN-roadmap-3stepのレイアウトで、STEP1〜3の期間［各ステップの期間］、内容［各ステップで行うこと］、成果物［各ステップのアウトプット］を入れてください。デザインはスライドマスターに従ってください。」

**注意点：**
- ステップ数は3が基本。4ステップ以上にしたい場合は横幅が窮屈になるため、カード幅を狭めるか2段組みへの変更を検討する。
- 強調するステップ（濃色ヘッダー）は基本的にSTEP1（今まさに着手するステップ）に限定する。全ステップを強調すると意図が伝わらない。
- 成果物欄は必ず具体的な名詞（文書名・数値目標等）で書く。「進捗」のような抽象語だけだとスライドの説得力が下がる。
