# SLIDE-PATTERN: org-chart-tree

## Overview

最上位ノード（役員会等）から部門、役職・業務リストへと階層的に分岐するツリー型組織図。垂直線と水平線で接続する。

## 適したシーン

会社・部署・プロジェクトの組織体制、意思決定構造、チームの役割分担を階層的に示すとき。

## Structure

```yaml
layout: tree
zones:
  - role: level-1
    content: 最上位ノード ×2（取締役会・監査役会等の最高意思決定機関）
  - role: level-2
    content: 部門ノード ×3〜5（CEO直下・各事業部門。視覚的に強調するノード）
  - role: level-3
    content: 部門ごとの役職・業務リスト（縦積み、4〜6項目が最適）
reading_order: 上から下（第1階層→第2階層→第3階層）。第2階層は左から右
notes: ノード間は垂直線（親子関係）・水平線（兄弟関係）で接続する。最上位ノードは2つ並べる場合（取締役会＋監査役会）が多い。
```

## Elements

| 要素 | 配置 | 役割 |
|---|---|---|
| 最上位ノード | 第1行・上部 | 取締役会・監査役会等の最高意思決定機関 |
| 部門ノード | 第2行・中部 | CEO直下・各事業部門のノード（視覚的に強調するノード） |
| 役職・業務リスト | 第3行・下部 | 各部門に所属する役職・業務名の縦リスト |
| 接続線 | ノード間 | 垂直線（親子）・水平線（兄弟）で階層を表現 |
| ロゴ | 左下 | 会社ロゴ・クレジット（省略可） |

## Usage Guide

- 第2行の部門数は3〜5が適切。6以上は横幅が狭くなる
- 各部門の役職リストは4〜6項目が最適
- 最上位ノードは2つ並べる場合（取締役会＋監査役会）が多い
- 部門ノードは他の階層より視覚的に強調する（配色はマスターに従う）
