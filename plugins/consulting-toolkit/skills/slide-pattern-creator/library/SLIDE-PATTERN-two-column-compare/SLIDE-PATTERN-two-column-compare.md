# SLIDE-PATTERN-two-column-compare

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスターと組み合わせてAIツールに渡すことで、このパターンのスライドを生成できます。タイトルエリア・ページ番号・装飾はスライドマスター（branded-pptx / html-artifactテーマ）で定義されるため、このファイルには含みません。プレビュー .html は生成当時の見本であり非規範。見た目は選択マスターが決めます。

## Overview

**パターン名：** two-column-compare
**概要：** コンテンツエリアを左右2カラムに分割し、それぞれに小見出し＋箇条書きまたはハイライト回答を配置する対比・比較レイアウト。右カラムのトップに強調した「答え」を1文置き、その下に詳細箇条書きを展開する構成が特徴。
**適したシーン：** 背景と目的、課題と解決、Before/After、現状と目指す姿など、2つの視点を並列で提示するスライド。特に「左：状況説明」「右：主張の結論＋根拠」の非対称対比に向く。

## Structure（構造）

コンテンツエリアを縦の点線または細線で左右2カラム（約48%：48%）に分割する。各カラム上部に小見出し（H3相当）を配置し、その下にコンテンツを縦に並べる。右カラムの上部には強調したキーメッセージを1〜2行置き、その下に補足する箇条書きを続ける。

```yaml
layout: two-column
zones:
  - role: left-column
    width: 48%
    content: 小見出し＋箇条書き 2〜3項目（各項目は本文＋補足サブテキスト）
  - role: divider
    content: 縦の区切り線（中央）
  - role: right-column
    width: 48%
    content: 小見出し＋強調キーメッセージ（1〜2行）＋箇条書き 2〜3項目
reading_order: 左 → 右
notes: 右カラムの強調キーメッセージは1〜2行に収める。左右の項目数の差は1〜2項目までに抑える。
```

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| 左カラム見出し（H3） | 左カラム上部 | 対比の一方の観点（例：背景／現状／Before）を提示 |
| 左カラム本文 | 左カラム中央 | 箇条書き＋補足サブテキストで状況・事実を並べる |
| 右カラム見出し（H3） | 右カラム上部 | 対比のもう一方の観点（例：目的／目指す姿／After）を提示 |
| 右カラム強調テキスト | 右カラム上部（見出し直下） | このスライドの結論・キーメッセージを強調 |
| 右カラム本文 | 右カラム中央 | 強調テキストの根拠となる箇条書き |
| 中央区切り線 | 2カラムの間 | 対比を視覚的に区切る細い点線 |

## Usage Guide（AIへの使い方）

このパターンをAIに指示する際のプロンプト例：

> 「SLIDE-PATTERN-two-column-compareのレイアウトで、左に「背景」の箇条書き3項目、右に「目的」のキーメッセージ1行＋根拠3項目を配置してください。デザインはスライドマスターに従ってください。」

**注意点：**
- 左右で行数を極端にアンバランスにしない。1〜2項目差までに収める
- 右カラムの強調テキストは1〜2行に抑える。長すぎると強調効果が薄れる
- 左右の見出しは同じ抽象度・同じ文字数感で対比を作る（例：「背景」と「目的」、「現状」と「目指す姿」）
