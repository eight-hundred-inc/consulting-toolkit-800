# SLIDE-PATTERN-staircase-roadmap-with-icons

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスターと組み合わせてAIツールに渡すことで、このパターンのスライドを生成できます。タイトルエリア・ページ番号・装飾はスライドマスター（branded-pptx / html-artifactテーマ）で定義されるため、このファイルには含みません。プレビュー .html は生成当時の見本であり非規範。見た目は選択マスターが決めます。

## Overview

**パターン名：** staircase-roadmap-with-icons
**概要：** 左下から右上に向かって階段状に積み上がるステップ図。各ステップの上にアイコン付き丸バッジとステップ番号・ラベルを配置し、右方向への進行（矢印）と上昇イメージを同時に表現するロードマップレイアウト。
**適したシーン：** 工程・ステップの進行順序の説明、ロードマップ紹介、プロセスフローの可視化、成長や進化の段階を視覚的に示したいスライド。

## Structure（構造）

コンテンツエリアの上部にリード文（1〜2行）を置き、その下にステップ図エリアを配置する。ステップ図は左下を起点に右上へと段差を持って並ぶ4段の階段形状で構成され、各ステップにはアイコンバッジ（丸形）・ステップ番号・ステップ名ラベルが付く。最終ステップの先に右向き矢印（ゴール方向）を添える。右下にはサブタイトルやシリーズ名のテキストを添えられる。

```yaml
layout: staircase-flow
zones:
  - role: lead
    content: リード文（1〜2行）
  - role: staircase
    elements:
      - ステップブロック ×4（推奨3〜5、左下→右上の階段形状）
      - ステップ番号（01、02…）
      - 丸形アイコンバッジ
      - ステップ名ラベル
      - 最終ステップの右に進行方向を示す矢印
  - role: sub_caption
    content: 右下のサブタイトルテキスト（任意）
reading_order: 上（リード文）→ 中央（階段：左下→右上）→ 右下（サブキャプション、任意）
notes: 階段の段差は全体の15〜20%程度を確保する。ステップ数は4段が標準（3段は余白を広く、5段は幅を狭めて調整）。アイコンバッジは階段ブロックに一部重ねる配置にする。
```

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| リード文 | ステップ図の上 | 全体のプロセスを1〜2文で説明する |
| 階段ブロック | 中央エリア | 左下から右上へ段差を持って並ぶステップの土台 |
| アイコンバッジ | 各ステップの上（丸形） | そのステップを象徴するアイコンを円形バッジ内に表示 |
| ステップ番号 | ステップラベルの上 | 「01：」「02：」等の連番でステップの順序を示す |
| ステップラベル | アイコンの上のテキスト | ステップの名称・工程名を短く表す |
| 右向き矢印 | 最終ステップの右 | 先への進行・ゴールへの方向性を示す |
| サブキャプション | 右下 | スライドのシリーズ名・補足タイトルを添える（任意） |

## Usage Guide（AIへの使い方）

このパターンをAIに指示する際のプロンプト例：

> 「SLIDE-PATTERN-staircase-roadmap-with-iconsのレイアウトで、4段階のプロセスを左下から右上の階段状に並べたロードマップスライドを作成してください。デザインはスライドマスターに従ってください。」

**注意点：**
- 階段の段差は視覚的にわかりやすい高さ（全体の15〜20%程度）を確保する。
- ステップ数は4段が標準。3段なら余白を広く、5段なら幅を狭めて調整する。
- アイコンバッジは丸形で、ステップの土台（階段ブロック）の上に配置し、一部重なるようにするとデザイン的にきれいになる。
- 最終ステップには矢印を付け「さらなる前進」のイメージを持たせる。
- 階段ブロックは単色（ブランドカラー）で構成する。
