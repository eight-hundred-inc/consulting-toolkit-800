# SLIDE-PATTERN-references-table

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスター（表紙・タイトル行・フッター・配色・フォント）はマスター側（branded-pptx / html-artifactテーマ等）で定義されるため、このファイルには含みません。プレビュー .html は生成当時の見本であり非規範。見た目は選択マスターが決めます。

## Overview
**パターン名：** references-table
**概要：** 番号・発行元・資料名・時期・URL・種別の6列からなる出典一覧テーブルを8〜10行並べ、下部に確認注記を添えるレイアウト。
**適したシーン：** 提案書や調査レポートの巻末に、本文で使用した一次・準一次情報源をまとめて開示するスライド。
**対応する図解指名：** 出典テーブル
**下流実装ヒント：** html-artifact: References Table ／ pptx: 6列テーブル（罫線のみ、塗りなし）＋下部注記テキストボックス

## Structure（構造）
コンテンツエリアは1本の横長テーブルで構成される。ヘッダー行に列名（番号・発行元・資料名・発行時期・URL・種別）を配置し、その下に8〜10行のデータ行を並べる。1つの出典が複数の関連URLを持つ場合はセル内でさらに小分けする、または行を結合してもよい。テーブルの下部には全体を通した確認注記（例：情報の確認日・照合状況）を1〜2行の小さなテキストで添える。

```yaml
layout: single-table-with-footnote
zones:
  - role: table
    content: 6列テーブル（番号・発行元・資料名・発行時期・URL・種別）× 8〜10行
  - role: footnote
    content: 確認注記（確認日・照合状況等、1〜2行）
reading_order: テーブル（列は左→右、行は上→下） → 確認注記
notes: 1出典に複数URLがある場合はセル内で小分けする、または行を結合して番号・発行元セルを共有する
```

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| テーブルヘッダー | テーブル最上部 | 各列が何の情報かを示す見出し行 |
| 出典番号列 | テーブル左端 | 本文中の脚注番号（※1等）とテーブル行を対応づける |
| 発行元列 | 番号列の右隣 | 情報の発信主体を示し、信頼性判断の材料にする |
| 資料名列 | 発行元列の右隣 | トピックの要約と正式な資料タイトルを示す |
| 発行時期列 | 資料名列の右隣 | 情報の鮮度・調査対象範囲を示す |
| URL列 | 発行時期列の右隣 | 一次情報への到達手段を示す（複数ある場合は名称付きで列挙） |
| 種別列 | テーブル右端 | 一次情報か準一次情報か等、情報源の性質を分類する |
| 確認注記 | テーブル直下 | いつ・どの範囲まで情報の正確性を確認したかを明示する |

## Usage Guide（AIへの使い方）

> 「SLIDE-PATTERN-references-tableのレイアウトで、出典一覧［各出典の発行元・資料名・時期・URL・種別］を8〜10件入れ、下部に確認注記［確認日・照合範囲］を入れてください。デザインはスライドマスターに従ってください。」

**注意点：**
- 資料名・URL列は文字量が多くなりやすいため、フォントサイズを他列より小さくして1行に収める。収まらない場合は資料名の前に短いトピックラベルを置いて情報を階層化する。
- 1つの出典に複数URL（複数の事例等）がある場合は、行を結合して番号・発行元セルを共有し、URLセル内のみを名称付きで縦に並べる。
- 行数が10を超える場合はテーブルが縦に窮屈になるため、優先度の低い出典を割愛するか2スライドに分割する。
