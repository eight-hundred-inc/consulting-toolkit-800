## Overview
**パターン名：** stacked-bar-hero-numbers
**概要：** 左2/3に構成比を示す横積み上げ棒グラフ、右1/3に大きな数値2つ（Hero Number）を配置し、下部に帰結コールアウト帯を敷く2カラム構造。
**適したシーン：** 業界構造・コスト構成など「内訳（構成比）」と「際立った実態値（絶対値）」を1枚で並べて見せ、両方から同じ結論を導きたいスライド。
**対応する図解指名：** 積み上げ棒＋Hero Number
**下流実装ヒント：** html-artifact: Hero Number＋図版 ／ pptx: shape棒＋大数字

## Structure（構造）
コンテンツエリアは左右2カラム（左約60%・右約40%、境界に縦罫線）で分割される。左カラムは「構成比セクション」と「補助データセクション」を縦に積み、構成比セクションは最大セグメントを示すブラケット注記→横積み上げ棒→凡例→補足説明文の順に並ぶ。右カラムはHero Number（大きな数値＋ラベル＋注記）を2つ縦に並べ、末尾に短い注記を置く。最下部には左右カラムをまたいで帰結コールアウト帯（結論ボックス＋示唆ボックスの2分割）を配置する。

```yaml
layout: two-column-with-bottom-band
zones:
  - role: intro
    content: 導入文（このスライドで何を確認するかを1文で示す前置き）
  - role: left_column
    width: 60%
    content: 構成比セクション（ブラケット注記→横積み上げ棒 3〜4セグメント→凡例→補足文）＋補助データブロック（見出しラベル＋大きい数値＋補足テキスト）
  - role: right_column
    width: 40%
    content: Hero Number ×2（ラベル＋大きな数値＋単位）＋注記
  - role: bottom_band
    content: 帰結ボックス（結論文＋根拠）／示唆ボックス（読み手への当てはめ・次の一手）の2分割
reading_order: 導入文 → 左カラム（ブラケット注記→積み上げ棒→凡例→補足文→補助データ） → 右カラム（Hero Number×2→注記） → 下部帯（帰結→示唆）
notes: 左右カラムの境界に縦罫線を引く。帰結ボックスと示唆ボックスは視覚的に区別する。積み上げ棒のセグメントは3〜4個までに絞り、5個を超える場合は「その他」でまとめる
```

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| 導入文 | コンテンツエリア最上部 | このスライドで何を確認するかを1文で示す前置き |
| ラベルチップ | 左カラム各セクション冒頭 | セクションの見出し（黒地に白文字の短いタグ） |
| ブラケット注記 | 積み上げ棒の直上 | 最大セグメントの構成比をブラケットで視覚的に強調する数値表示 |
| 横積み上げ棒 | 左カラム中央 | 全体を100%として各セグメントの構成比を面積で示す |
| 凡例 | 積み上げ棒の直下 | 各セグメントの名称と数値を色見本付きで一覧化 |
| 棒グラフの補足文 | 凡例の直下 | 最大セグメントが突出している事実を短く言語化 |
| 補助データブロック | 左カラム下部 | 構成比とは別の指標（比率・水準等）を大きい数値＋補足で提示 |
| Hero Number×2 | 右カラム | 際立った実態値を、ラベル・大きな数値・単位のセットで2つ提示 |
| 右カラム注記 | 右カラム最下部 | Hero Numberを補強する追加のファクト |
| 帰結ボックス | 下部帯・左 | 左右カラムの情報から導かれる結論を明言する |
| 示唆ボックス | 下部帯・右 | 結論を読み手（クライアント等）の状況に当てはめた一言 |

## Usage Guide（AIへの使い方）

> 「SLIDE-PATTERN-stacked-bar-hero-numbersのレイアウトで、左カラムに［対象の構成比データ・最大セグメント名・比率］、右カラムに［際立った実態値2つとその単位・意味］を入れ、下部の帰結欄に［この2つのデータから導かれる結論と示唆］を記入してください。デザインはスライドマスターに従ってください。」

**注意点：**
- 積み上げ棒のセグメントは3〜4個までに絞る。5個を超えると凡例が読みにくくなるため「その他」でまとめる。
- Hero Numberは2つが基本。3つ以上入れたい場合は右カラムの縦スペースが不足するため、KPI Row等の別パターンを検討する。
- 左カラムの構成比データと右カラムの実態値は、同じテーマ（例：コスト構造と非稼働実態）で対になっている必要がある。無関係な指標を並べると帰結が導けなくなる。
