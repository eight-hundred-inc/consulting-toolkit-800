# SLIDE-PATTERN-attribute-rows-profile

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスター（表紙・タイトル行・フッター・配色・フォント）はマスター側（branded-pptx / html-artifactテーマ等）で定義されるため、このファイルには含みません。

## Overview
**パターン名：** attribute-rows-profile
**概要：** 属性の行ラベル（例：Product／Place／Promotion）を縦に並べ、各行を「行ラベル×中央の画像×右の説明文」の3列で構成した、対象1件のプロファイル。
**適したシーン：** 1社・1ブランド・1製品を決まった観点（4Pなど）に沿って多面的に紹介するとき。ブランド概説、商品戦略、販売戦略などのプロファイルスライド。
**対応する図解指名：** 属性行プロファイル
**下流実装ヒント：** html-artifact: 行ラベル付き2カラム表（各行にラベル＋画像＋説明） ／ pptx: 行ラベル列＋画像＋テキストの3列表を行ごとに繰り返す

## Structure（構造）
1件の対象について、属性ごとに横長の行を縦に3〜4行繰り返す。各行は左に属性の行ラベル、中央に画像プレースホルダ、右に説明文を配置する。行ラベルは4Pなど固定の観点セットを使うと対象間の比較がしやすい。

```yaml
layout: repeating-attribute-rows
subject: single   # 対象は1件
areas:
  - name: attribute-row
    role: 属性ごとの行（繰り返し）
    repeat: 3-4
    columns:
      - attribute-label   # 左：属性の行ラベル（Product/Place/Promotion等）
      - image             # 中央：画像プレースホルダ
      - description       # 右：説明文
```

## Elements（各要素の役割）
| 要素 | 配置 | 役割 |
|------|------|------|
| 属性ラベル | 各行の左 | その行が扱う観点（Product/Price/Place/Promotion等）を示す |
| 画像プレースホルダ | 各行の中央 | 製品写真・画面・ロゴなど属性を象徴するビジュアル |
| 説明文 | 各行の右 | その属性についての要点を2〜3行で説明する |

## Usage Guide（AIへの使い方）
プロンプト例：「対象ブランド1社を4Pのうち Product／Place／Promotion の3観点でプロファイルにして。各行は左に観点ラベル、中央に該当ビジュアルの枠、右に2〜3行の説明。」

注意点：
- 1スライド＝1対象。複数対象を比較したい場合は comparison-matrix を使う。
- 行ラベルの観点セットはスライド間で揃える（対象ごとに観点が変わると比較できない）。
- 画像が用意できない行は中央列をプレースホルダのまま残すか、説明文を中央まで広げて2列構成に落とす。
