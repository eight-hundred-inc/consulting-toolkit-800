# SLIDE-PATTERN-numbered-toc

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスターと組み合わせてAIツールに渡すことで、このパターンのスライドを生成できます。タイトルエリア・ページ番号・装飾はスライドマスター（branded-pptx / html-artifactテーマ）で定義されるため、このファイルには含みません。プレビュー .html は生成当時の見本であり非規範。見た目は選択マスターが決めます。

## Overview

**パターン名：** numbered-toc
**概要：** 番号・項目名・点線・ページ番号を横一列に並べたリスト行を縦積みで表示する目次レイアウト
**適したシーン：** プレゼンテーション・提案書の目次スライド、章立てが明確なドキュメントの構成提示

## Structure（構造）

コンテンツエリア全体に縦積みのリストを中央揃えで配置する。各行は「番号. 項目名」「点線（ドットリーダー）」「ページ番号」の3要素が横並びになっており、行間に十分な余白を持たせて視認性を高める。

```yaml
layout: single
zones:
  - role: toc-list
    content: 目次行 ×4〜8（項目数に応じて調整、中央〜左寄せで配置）
    elements:
      - role: number
        content: 連番（「1.」「2.」形式）
      - role: item-name
        content: 章・セクション名
      - role: dotted-leader
        content: 項目名とページ番号をつなぐ点線
      - role: page-number
        content: 開始ページ番号
reading_order: 上から下（行順）。各行内は左（番号→項目名）→ 右（点線→ページ番号）
notes: 行間に余白を設けてゆったり表示する
```

## Elements（各要素の役割）

※ タイトル（H1）はスライドマスターの Slide Frame で定義されるため、この表には含めない。コンテンツエリアの要素のみを記述する。

| 要素 | 配置 | 役割 |
|---|---|---|
| 番号 | 各行・左端 | 目次項目の連番（「1.」「2.」形式） |
| 項目名 | 各行・番号の右 | 章・セクションの名称（日本語または英語） |
| 点線（ドットリーダー） | 各行・項目名とページ番号の間 | 項目名とページ番号を視覚的に繋ぐ装飾線 |
| ページ番号 | 各行・右端 | 該当セクションの開始ページ番号 |

## Usage Guide（AIへの使い方）

このパターンをAIに指示する際のプロンプト例：

> 「SLIDE-PATTERN-numbered-tocのレイアウトで、[項目名とページ番号のリスト]を使って目次スライドを作成してください。デザインはスライドマスターに従ってください。」

**注意点：**
- 点線（ドットリーダー）は罫線・テキスト記号のいずれの表現でもよい
- 番号と項目名の間のスペースを揃えることで整列感が出る
- 項目数が多い場合（8行以上）は2カラム構成への変更を検討する
- 現在のセクションを強調色で表示するバリエーションも有効
