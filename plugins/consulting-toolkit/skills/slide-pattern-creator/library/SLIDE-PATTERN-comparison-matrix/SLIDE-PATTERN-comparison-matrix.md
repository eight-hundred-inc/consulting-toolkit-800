# SLIDE-PATTERN-comparison-matrix

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスター（表紙・タイトル行・フッター・配色・フォント）はマスター側（branded-pptx / html-artifactテーマ等）で定義されるため、このファイルには含みません。プレビュー .html は生成当時の見本であり非規範。見た目は選択マスターが決めます。

## Overview
**パターン名：** comparison-matrix
**概要：** 列に比較対象（N社・N案）、行に比較観点（M項目）を取り、セルにテキストまたは記号評価を入れる比較マトリクス。
**適したシーン：** 複数の対象を同じ観点で横並び比較するとき。競合ベンチマーク、3社比較、選択肢の評価一覧など。1つの対象を推す場合はその列を強調する。
**対応する図解指名：** 比較マトリクス
**下流実装ヒント：** html-artifact: Report Table ＋ Rating Dots ／ pptx: ネイティブ表（列ヘッダに対象名、1列を強調、評価セルは記号または評価ドット）

## Structure（構造）
表の左上は空セル、上端ヘッダ行に比較対象を横に並べ、左端列に比較観点を縦に並べる。各セルはテキストの短評、または記号／評価ドット（●○）で表す。推奨・注目する対象がある場合は、その列のヘッダとセルを1列だけ強調する。

```yaml
layout: matrix-table
zones:
  - role: lead
    content: 全幅の要約文（1文・任意）
    optional: true
  - role: comparison-matrix
    content: 比較マトリクス本体（列ヘッダー＝比較対象、行ヘッダー＝比較観点、セルは短評テキストまたは評価ドット）
    column_headers: targets（比較対象）
    row_headers: perspectives（比較観点）
    target_count: 2-4
    perspective_count: 3-6
    cell_style: text-or-rating
    emphasis: single-column（任意、1列のみ強調可）
reading_order: 上（リード文）→ マトリクス（列ヘッダー → 行ヘッダー → 各セル）
notes: 比較対象は2〜4列、観点は3〜6行に収める。強調列は多くても1列
```

## Elements（各要素の役割）
| 要素 | 配置 | 役割 |
|------|------|------|
| 空セル | 左上隅 | 行ラベル列とヘッダ行の交点（無地） |
| 対象ヘッダ | 上端ヘッダ行 | 比較する対象（社・案）名を横に並べる |
| 観点ラベル | 左端列 | 比較する観点を縦に並べる |
| 評価セル | 本体の各セル | 対象×観点の評価を短評または評価ドットで示す |
| 強調列 | 対象ヘッダの1列 | 推奨・注目する対象を1列だけ強調する（任意） |

## Usage Guide（AIへの使い方）
プロンプト例：「3社を『特性・強み・課題・総合評価』の4観点で比較するマトリクスにして。列ヘッダに社名、総合評価は5段階の評価ドットで。自社が推す1社の列を強調して。」

注意点：
- 比較対象は2〜4列、観点は3〜6行に収める。多すぎる場合は観点を統合する。
- 評価は「テキスト短評」と「評価ドット」を混在させてよいが、同じ行内では表現を揃える。
- 強調列は多くても1列。全列を強調すると比較の主張が消える。定量評価データが無い観点はテキスト短評で代替する。
