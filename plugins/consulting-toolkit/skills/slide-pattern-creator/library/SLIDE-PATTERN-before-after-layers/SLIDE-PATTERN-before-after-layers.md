# SLIDE-PATTERN-before-after-layers

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスター（表紙・タイトル行・フッター・配色・フォント）はマスター側（branded-pptx / html-artifactテーマ等）で定義されるため、このファイルには含みません。

## Overview
**パターン名：** before-after-layers
**概要：** 上段にBefore→After 2パネル、中段に実装構成のレイヤー積層（4〜5層）、下段に類型カード（2×2）＋展開強調帯を縦に重ねる3段構造。
**適したシーン：** 施策の全体像を「現状から目指す姿への変化」「それを支える実装構成」「実装後に生まれるパターンの横展開」という3層の粒度で1枚に通しで説明したいスライド。情報量が多い施策詳細の解説に向く。
**対応する図解指名：** Before-After＋レイヤー＋類型
**下流実装ヒント：** html-artifact: State Grid＋レイヤー積層＋カードグリッド ／ pptx: 2パネル＋層状シェイプ＋2×2カード

## Structure（構造）
コンテンツエリアは上から「今の状態→目指す姿」「実装構成（レイヤー積層）」「類型カード＋展開」の3セクションを縦に並べる。各セクションの先頭にはセクション見出し＋横罫線を置く。第1セクションはBeforeパネルとAfterパネルを矢印でつなぐ2カラム。第2セクションは4〜5層の横長バーを縦に積み上げ、各層に番号・層名・説明を横並びで記載する。第3セクションは左に2×2の類型カードグリッド、右に矢印でつながる強調帯（黒背景等）を配置し、類型を実務にどう展開するかを説明する。セクション末尾に先行事例の注記帯を置いてもよい。

    structure:
      layout: three-section-stack
      section_1:
        label: 今の状態 → 目指す姿
        panels:
          before: { tag: 今, body: 現状の説明 }
          after: { tag: 目指す姿, body: 変化後の状態の説明, emphasis: 太罫線で強調 }
          connector: arrow
      section_2:
        label: 実装構成（N層）
        layers:
          count: 4-5
          each: { index: 層番号, name: 層名, desc: 説明文 }
      section_3:
        label: 類型の分解 → 横展開
        left:
          intro: 類型分解の説明文
          grid: 2x2
          each_card: { title: 類型名, body: 説明文 }
        right:
          expansion_band:
            style: 塗りつぶし強調（黒背景等）
            label: 展開
            body: 類型をどう標準化・横展開するかの説明
      footer_note:
        tag: 先行事例
        body: 根拠となる事例（任意）

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| セクション見出し×3 | 各セクション先頭 | 「現状→目指す姿」「実装構成」「類型→展開」の3段階を明示する |
| Beforeパネル | 第1セクション左 | 現状の課題・状態を説明する |
| Afterパネル | 第1セクション右 | 施策実行後に目指す状態を説明する（枠線を太くする等で視覚的に強調） |
| 矢印コネクタ | Before/Afterの間 | 現状から目指す姿への変化・移行を示す |
| レイヤー（4〜5層） | 第2セクション | 施策を実現するための実装構成要素を、下位層から上位層（または処理順）に積み上げて示す |
| 類型カード（2×2） | 第3セクション左 | 実装後に観測される複数のパターン・類型を4分割で提示する |
| 展開強調帯 | 第3セクション右 | 類型カードの内容を実務でどう横展開・標準化するかをまとめて強調する |
| 先行事例の注記 | セクション末尾（任意） | 施策の実現可能性を裏付ける事例・実績への言及 |

## Usage Guide（AIへの使い方）

> 「SLIDE-PATTERN-before-after-layersのレイアウトで、上段に［現状と目指す姿の対比］、中段に［実装を支える4〜5層の構成要素］、下段に［実行後に見られる4類型とその横展開方法］を記入してください。デザインはスライドマスターに従ってください。」

**注意点：**
- 3セクション構成のため情報量が非常に多くなりやすい。各セクションの本文は1〜2文に圧縮し、詳細な説明が必要な場合は別スライドに分割することを検討する（特に第2セクションのレイヤーは層ごとの説明を簡潔に保つ）。
- レイヤーの数は4〜5層を上限とする。6層を超える場合は上位レイヤーと下位レイヤーでスライドを分ける。
- 類型カードは2×2の4枚を標準とする。3枚以下や5枚以上にしたい場合はグリッド比率が崩れるため、カード数を4枚に調整するか別パターン（カード・グリッド系）を検討する。
