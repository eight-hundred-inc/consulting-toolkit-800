# SLIDE-PATTERN-cycle-diagram-annotated

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスター（表紙・タイトル行・フッター・配色・フォント）はマスター側（branded-pptx / html-artifactテーマ等）で定義されるため、このファイルには含みません。プレビュー .html は生成当時の見本であり非規範。見た目は選択マスターが決めます。

## Overview
**パターン名：** cycle-diagram-annotated
**概要：** コンテンツエリア中央に核となるメッセージを置いた円環（4〜6ノード）のサイクル図を配置し、その周辺に各ノードと対応した注記ボックス（問い・説明）を並べる構造。
**適したシーン：** 中心テーマを取り巻く複数の観点・問い・構成要素を、循環・全方位的な関係として示したいスライド。「全体像」「検討の論点マップ」「継続的サイクル」を1枚で俯瞰させたい場面に向く。
**対応する図解指名：** サイクル図＋注記
**下流実装ヒント：** html-artifact: サイクル図 ／ pptx: 円環shape＋注記ボックス

## Structure（構造）
コンテンツエリア中央に、中核メッセージを収めたハブ円を中心とし、その周囲に4〜6個のノード円を円環状に配置したサイクル図を置く。左右（または四隅）の余白に、各ノードと対応した注記ボックスを並べ、ノード番号や近接配置でノードと注記を対応づける。注記には各ノードに対する問い・説明を1〜2文で記す。ノードは循環関係（順に巡る／相互に関連する）を示す並びとする。

```yaml
layout: cycle-diagram-annotated
zones:
  - role: hub
    content: 中核メッセージ（円の中心）
  - role: ring
    content: ノード円 4〜6個（円環状・等間隔、循環・全方位的な関係）
  - role: annotations
    content: 注記ボックス（中央図の左右または四隅、各ノードに対応する問い・説明を1〜2文）
reading_order: 中央（ハブ）→ 周囲のノード（円環順）→ 対応する注記
notes: ノードと注記は番号または近接配置で1対1に対応させる
```

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| ハブ円 | 中央 | サイクル全体が向かう中核メッセージ・目的を1点に集約して示す |
| ノード円 | ハブ周囲の円環状 | サイクルを構成する各要素・段階・観点。4〜6個で全体を分割する |
| 注記ボックス | 中央図の左右／四隅 | 各ノードに対応する問い・説明を記し、ノードの意味を言語化する |
| 対応づけ | ノード⇔注記 | 番号または近接配置でどの注記がどのノードのものか示す |

## Usage Guide（AIへの使い方）

> 「SLIDE-PATTERN-cycle-diagram-annotatedのレイアウトで、中央ハブに［中核メッセージ］、周囲のノードに［観点1〜観点N（4〜6個）］を置き、各ノードに対応する注記に［そのノードに対する問い・説明］を入れてください。デザインはスライドマスターに従ってください。」

**注意点：**
- ノードは4〜6個が適切。3個以下ならサイクル感が出ず、7個以上だと円環が窮屈になり注記も配置しきれない。
- ノードと注記の対応は番号（①②…）または明確な近接配置で示す。対応が曖昧だと読み手がどの注記がどのノードのものか追えない。
- サイクル（循環）の意味が無い並列要素の場合は、円環ではなく別パターン（番号リスト等）を使う。
