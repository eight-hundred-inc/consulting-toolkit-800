# SLIDE-PATTERN-recognition-cards-issues

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスター（表紙・タイトル行・フッター・配色・フォント）はマスター側（branded-pptx / html-artifactテーマ等）で定義されるため、このファイルには含みません。

## Overview
**パターン名：** recognition-cards-issues
**概要：** 上段に認識合わせカード4枚を横並びで配置し、下段にヘッダ帯付きの論点カード2枚を配置する、上段→下段への絞り込み構造。
**適したシーン：** 提案・議論の冒頭で「前提となる理解（4点程度）」を確認したうえで、そこから導かれる「絞り込んだ論点（2つ）」へつなげたいスライド。プロジェクト冒頭のキックオフ・現状整理に向く。
**対応する図解指名：** 認識カード＋論点絞り込み
**下流実装ヒント：** html-artifact: カード4枚グリッド＋Proposal Card 2枚 ／ pptx: 4枚カードグリッド＋2枚ヘッダ付きカード

## Structure（構造）
コンテンツエリア上部にセクション見出し（＋補足ラベル）を置き、その下に4枚の認識合わせカードを2×2グリッドで配置する（既定。横長要素で1行に収める意図が構成側で明示された場合のみ1×4も可）。各カードは短いラベルチップと本文で構成される。カード群の下に「論点は2つに絞れる」等の帯（強調バンド）を挟み、その下に論点カード2枚を横並びで配置する。論点カードはヘッダ帯（見出し）と本文で構成され、上段の4カードから論点が絞り込まれたことが視覚的に伝わる構造にする。末尾に次スライドへの橋渡し文を置いてもよい。

    structure:
      layout: two-tier-narrowing
      section_1:
        heading: 認識合わせ（4点 等）
        note: 補足ラベル（任意。例：ズレがあれば指摘してほしい旨）
        grid: 2x2（既定。4カードは2×2が均整。1x4は横長要素の明示指定時のみ）
        cards:
          count: 4
          each: { chip: 観点ラベル, body: 説明文 }
      transition_band:
        style: 塗りつぶし強調（黒背景等）
        body: 論点が絞り込まれたことを示す一言
      section_2:
        grid: 1x2
        cards:
          count: 2
          each:
            header: 論点名（論点A／論点B 等）
            body: 論点の説明文
      bridge_note: 次スライドへの橋渡し文（任意）

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| セクション見出し | 上段先頭 | 「認識合わせ」等、これから確認する内容の見出し |
| 補足ラベル | セクション見出しの隣 | 読み手への一言添え書き（任意） |
| 認識カード（4枚） | 上段グリッド | 前提となる理解を観点別（事業構造・体制・コスト等）に4つ提示する |
| 観点ラベルチップ | 各認識カード上部 | カードが扱う観点を短いタグで明示 |
| 絞り込み帯 | 上段と下段の間 | 4つの認識から論点が絞り込まれたことを視覚的・文言的に示す転換点 |
| 論点カード（2枚） | 下段 | 絞り込んだ論点を、ヘッダ＋本文のセットで2つ提示する |
| 論点カードヘッダ | 各論点カード上部 | 論点の名称（論点A／論点B 等）を明示 |
| 橋渡し文 | コンテンツエリア最下部（任意） | この論点が次スライド以降でどう検証されるかを予告する |

## Usage Guide（AIへの使い方）

> 「SLIDE-PATTERN-recognition-cards-issuesのレイアウトで、上段の認識カード4枚に［前提となる理解4点：観点名と説明］、下段の論点カード2枚に［そこから絞り込んだ論点2つ：論点名と説明］を記入してください。デザインはスライドマスターに従ってください。」

**注意点：**
- 上段のカード数は4枚が標準。3枚以下や5枚以上にすると2×2グリッドの均整が崩れるため、4枚に集約するか別パターン（カードグリッド系）を検討する。
- 下段の論点カードは、上段4カードの内容と論理的につながっている必要がある。上段と無関係な論点を下段に置くと「絞り込み」の構造が成立しなくなる。
- 情報量が多く1枚に収まらない場合は、上段の認識合わせを独立したスライド（カード4枚グリッドのみ）に分割し、本パターンは論点提示に絞ることを検討する。
