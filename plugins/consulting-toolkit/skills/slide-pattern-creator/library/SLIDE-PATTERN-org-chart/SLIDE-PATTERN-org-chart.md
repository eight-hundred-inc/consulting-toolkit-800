# SLIDE-PATTERN-org-chart

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスターと組み合わせてAIツールに渡すことで、このパターンのスライドを生成できます。タイトルエリア・ページ番号・装飾はスライドマスター（branded-pptx / html-artifactテーマ）で定義されるため、このファイルには含みません。プレビュー .html は生成当時の見本であり非規範。見た目は選択マスターが決めます。

## Overview

**パターン名：** org-chart
**概要：** プロジェクト体制を上下3階層（プロジェクト責任者→PM→メンバー）の縦型階層図で示すシンプルな体制図。各階層はヘッダー帯付きのボックスで複数名を横並びに表示し、階層間を縦線で接続する。
**適したシーン：** 提案書内のプロジェクト体制紹介、コンサル案件のアサインメンバー提示、案件責任者と実務担当者の階層表示。1ページ1体制で完結し、詳細な部門構造の表現は求められない場面に最適。

## Structure（構造）

コンテンツエリアを縦3段に分割し、各段に1つの階層ボックスを中央配置する。各ボックスは上部にヘッダー帯（役割名：プロジェクト責任者／プロジェクトマネージャー／プロジェクトメンバー）、下部に氏名エリア（役職＋氏名）を配置。ボックス間は縦線で接続し、階層関係を視覚化する。

```yaml
layout: vertical-3-tier-hierarchy
zones:
  - role: tier-1
    content: 責任者ボックス ×1（プロジェクト責任者、1〜2名を役職＋氏名で横並び表示）
  - role: tier-2
    content: PMボックス ×1（プロジェクトマネージャー、1名）
  - role: tier-3
    content: メンバーボックス ×1（プロジェクトメンバー、1〜3名）
reading_order: 上から下（責任者 → PM → メンバー）
notes: 各ボックスは上部にヘッダー帯（役割名を表示）、下部に氏名エリア（役職＋氏名）を持つ2段構成。階層間は縦線で接続する。
```

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| 責任者ボックス | 最上段 | プロジェクト全体の責任者を表示（1〜2名） |
| PMボックス | 中段 | プロジェクトマネージャーを表示（1名） |
| メンバーボックス | 最下段 | 実務メンバーを表示（1〜3名） |
| ヘッダー帯 | 各ボックス上部 | 役割名を表示する帯 |
| 役職ラベル | 各人の氏名の上 | 「マネージングディレクター」等の役職名 |
| 氏名 | 各人の中央 | フルネームを本文相当のテキストで表示 |
| 縦接続線 | ボックス間 | 階層関係を示す接続線 |

## Usage Guide（AIへの使い方）

このパターンをAIに指示する際のプロンプト例：

> 「SLIDE-PATTERN-org-chartのレイアウトで、プロジェクト体制を3階層（責任者2名／PM1名／メンバー1名）の縦型で表示してください。デザインはスライドマスターに従ってください。」

**注意点：**
- 階層は3段が最適。4段以上になる場合は org-chart-tree の使用を検討する
- 各階層の人数は1〜3名までに抑える。多いと横幅が窮屈になる
- 責任者・PM・メンバーは案件ごとの標準的な3階層構成に対応。役割名は柔軟に変更可
