# SLIDE-PATTERN-case-study-table

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスターと組み合わせてAIツールに渡すことで、このパターンのスライドを生成できます。タイトルエリア・ページ番号・装飾はスライドマスター（branded-pptx / html-artifactテーマ）で定義されるため、このファイルには含みません。

## Overview

**パターン名：** case-study-table
**概要：** 「企業名」「ポイント（要約）」「概要（詳細）」の3列で複数事例を縦に列挙するテーブル。企業ごとに複数のポイント行を持てるようセルが縦にマージされ、密度の高い事例比較を可能にする。
**適したシーン：** 先行企業事例調査、ベンチマーク比較、複数企業のベストプラクティスを一覧表示するスライド。1ページで4〜6社×2〜3ポイント程度の事例を凝縮して示したい場面に最適。

## Structure（構造）

コンテンツエリアを1枚のテーブルにする。列は「企業名（15%）」「ポイント（30%）」「概要（55%）」の3カラム。行は事例数×ポイント数。同一企業の複数ポイントは企業列をセル結合して表現する。ヘッダー行はText Mutedカラーの小さな英字。

    structure:
      layout: three-column-table
      columns:
        - name: "企業名"
          width: 15%
          style: "アクセントカラー（グリーン等）の太字"
        - name: "ポイント"
          width: 30%
          style: "アクセントカラー（ブルー等）の太字"
        - name: "概要"
          width: 55%
          style: "Text Subカラー・本文サイズ"
      row_grouping: "企業列は複数ポイントをまたいでセル結合"
      row_dividers: "1px dotted Border"

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| ヘッダー行 | テーブル最上部 | 列名（企業／ポイント／概要）をText Mutedカラーで表示 |
| 企業名セル | 各行の左列 | アクセントカラー1（例：グリーン）で企業名を強調。同一企業は結合 |
| ポイントセル | 中央列 | アクセントカラー2（例：ブルー）で要約タグを表示 |
| 概要セル | 右列 | 詳細説明と出典（※注釈番号）を1〜2行で記述 |
| 行区切り線 | 各行の下端 | 薄いグレー点線で行を視覚的に分離 |
| 出典注釈 | スライド最下部 | ※1〜※Nの出典を小さく列挙 |

## Usage Guide（AIへの使い方）

このパターンをAIに指示する際のプロンプト例：

> 「SLIDE-PATTERN-case-study-tableのレイアウトで、4社の先行事例をそれぞれ2〜3ポイントずつ列挙するテーブルを作ってください。企業名・ポイント・概要の3列構成で、出典は末尾に注釈で示してください。デザインはスライドマスターに従ってください。」

**注意点：**
- 情報量が多いテーブルなので、フォントサイズを標準本文より1〜2pt小さくする
- 「概要」列は事実ベースで簡潔に。解釈は「ポイント」列に凝縮する
- 出典は必ず※番号で明示し、原典に忠実に引用する（原則8：引用の忠実性）
