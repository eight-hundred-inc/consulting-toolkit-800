# SLIDE-PATTERN-status-progress-table

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスターと組み合わせてAIツールに渡すことで、このパターンのスライドを生成できます。タイトルエリア・ページ番号・装飾はスライドマスター（branded-pptx / html-artifactテーマ）で定義されるため、このファイルには含みません。プレビュー .html は生成当時の見本であり非規範。見た目は選択マスターが決めます。

## Overview

**パターン名：** status-progress-table
**概要：** プロジェクトや施策の一覧を表形式で示し、各行にステータスアイコン（完了/進行中/遅延など）と日程・備考を表示する進捗管理レイアウト
**適したシーン：** 月次レポートの進捗確認、プロジェクト一覧の状況報告、施策ステータスの共有

## Structure（構造）

コンテンツエリアにテーブルを配置する。列構成は「項目名」「ステータス（アイコン＋テキスト）」「完了日/状況」の3列が基本。行数は5〜8行程度。ステータス列にはアイコンとステータスごとに区別したテキストを使用する。

```yaml
layout: data-table
zones:
  - role: table
    content: テーブル（3列 × 5〜8行）
    columns:
      - 項目名（強調表示）
      - ステータス（アイコン＋テキスト）
      - 日付・備考
    status_values: [完了（チェックアイコン）, 進行中（スピナーアイコン）, 遅延（警告アイコン）]
reading_order: 上から下（行）／ 左から右（列）
notes: 行数は7行以内が可読性を保てる上限。ステータスアイコンは✓（完了）、○（進行中）、△（遅延/課題）など汎用記号でも表現できる。
```

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| 項目名 | 各行左列 | プロジェクト・施策名（番号付き） |
| ステータスアイコン | 各行中央列左 | 完了/進行中/遅延を視覚で即時識別 |
| ステータステキスト | アイコン右 | ステータス名称テキスト（色分け） |
| 日時・備考 | 各行右列 | 完了日、予定日、または状況の補足 |
| 行区切り線 | 各行間 | 行の境界を明確に示す |

## Usage Guide（AIへの使い方）

このパターンをAIに指示する際のプロンプト例：

> 「SLIDE-PATTERN-status-progress-tableのレイアウトで、[N]つのプロジェクトの進捗一覧表を作成してください。各行にプロジェクト名・ステータス（完了/進行中/遅延）・完了日を入れてください。デザインはスライドマスターに従ってください。」

**注意点：**
- ステータスアイコンは✓（完了）、○（進行中）、△（遅延/課題）など汎用的な記号でも表現できる
- 行数は7行以内が可読性を保てる上限
- 遅延行は背景色を薄くハイライトするとリスクが一目でわかる
