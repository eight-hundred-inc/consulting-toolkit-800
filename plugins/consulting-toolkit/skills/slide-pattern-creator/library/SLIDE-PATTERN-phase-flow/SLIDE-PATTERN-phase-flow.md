# SLIDE-PATTERN-phase-flow

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスター（表紙・タイトル行・フッター・配色・フォント）はマスター側（branded-pptx / html-artifactテーマ等）で定義されるため、このファイルには含みません。

## Overview
**パターン名：** phase-flow
**概要：** 横方向に3〜5個のフェーズをシェブロン（矢印形）またはボックスで並べ、各フェーズの配下にそのフェーズで実施するタスク・成果物を列挙する、時間・段階の進行を示すプロセスフロー。
**適したシーン：** プロジェクトの進め方・調査ステップ・導入プロセスなど、複数の段階を順に踏んで進む流れを示したいスライド。各段階で「何をやるか／何が出るか」まで見せたい場面に向く。
**対応する図解指名：** フェーズフロー
**下流実装ヒント：** html-artifact: 概念フロー（横） ／ pptx: シェブロン/矢印shape＋タスク列

## Structure（構造）
コンテンツエリア上段に、左から右へ進行を示すフェーズ帯（シェブロンまたはボックス）を3〜5個等間隔で並べる。矢印形または右向きの流れで段階の順序を明示する。下段には各フェーズと縦位置を揃えたタスクボックスを置き、そのフェーズで実施する作業・生成される成果物を箇条書きで列挙する。フェーズ帯とタスクボックスは列で対応させ、上のフェーズと下のタスクが1対1で結びつくようにする。

    structure:
      layout: phase-flow
      direction: horizontal
      phases:
        count: 3-5
        header:
          shape: chevron | box
          flow: 左から右への進行
          label: フェーズ名（例：フェーズ1 現状把握）
        body:
          shape: タスクボックス（フェーズ帯と縦位置を揃える）
          content: そのフェーズのタスク・成果物（箇条書き）
      alignment: 各フェーズ帯とその配下タスクボックスは列で対応させる

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| フェーズ帯 | 上段・横一列 | 各段階の名称。シェブロン/矢印形で左→右の進行順を示す |
| タスクボックス | 下段・各フェーズの直下 | そのフェーズで実施する作業・生成される成果物を箇条書きで示す |
| 進行方向の流れ | フェーズ帯間 | フェーズが順に進むこと（前段の完了が次段の前提）を矢印形状で表現する |

## Usage Guide（AIへの使い方）

> 「SLIDE-PATTERN-phase-flowのレイアウトで、フェーズ帯を［フェーズ1／フェーズ2／フェーズ3／フェーズ4］の4段階にし、各フェーズの配下に［そのフェーズのタスク・成果物］を箇条書きで入れてください。左から右への進行順で並べてください。デザインはスライドマスターに従ってください。」

**注意点：**
- フェーズは3〜5個が適切。6個以上になると1フェーズあたりの幅が狭くなりタスクが読めなくなるため、統合するか2スライドに分ける。
- フェーズ名は動作・段階が分かる名詞句（現状把握／詳細調査／提言 等）にし、単なる「フェーズ1」だけで終わらせない。
- 各フェーズのタスク数は揃える必要はないが、極端に偏る（1つのフェーズだけ10項目）場合は粒度を見直す。
