# SLIDE-PATTERN-swimlane-schedule

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスター（表紙・タイトル行・フッター・配色・フォント）はマスター側（branded-pptx / html-artifactテーマ等）で定義されるため、このファイルには含みません。

## Overview
**パターン名：** swimlane-schedule
**概要：** 縦にワークストリーム（担当・作業区分）、横に時間軸を取ったスイムレーン形式のスケジュール表。各レーンに期間バーとマイルストーン記号を配置して、いつ・誰が・何をするかを一望させる。
**適したシーン：** プロジェクトの実行計画・調査スケジュール・導入ロードマップなど、複数のワークストリームが時間軸上でどう並走・連携するかを示したいスライド。マイルストーンや期間の重なりを見せたい場面に向く。
**対応する図解指名：** スイムレーン・スケジュール
**下流実装ヒント：** html-artifact: Roadmap系 ／ pptx: 表＋バーshape＋マイルストーン記号

## Structure（構造）
コンテンツエリアを格子状に構成し、左端の列にワークストリーム（担当・作業区分）のラベルを縦に並べ、上端の行に時間軸（週・月・フェーズ等）の目盛りを置く。各レーン（行）の時間軸上に、作業期間を示す横バーと、節目を示すマイルストーン記号（◆等）を配置する。バーの開始・終了位置と長さで各作業のタイミングと期間を表現し、レーン間でバーが重なる箇所から並走関係を読み取れるようにする。

    structure:
      layout: swimlane-schedule
      axes:
        vertical: ワークストリーム（担当・作業区分のレーン）
        horizontal: 時間軸（週・月・フェーズ等の目盛り）
      grid:
        left_column: レーンラベル
        top_row: 時間軸ラベル
        cells: 時間軸上のトラック（バー・マイルストーンを配置）
      markers:
        bar: 作業期間（開始位置・長さで期間を表現）
        milestone: 節目の記号（◆等）
      readout: レーン間のバーの重なりから並走・連携を読み取る

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| レーンラベル | 左端の列・各行 | ワークストリーム（担当・作業区分）の名称。行が誰の・何の作業かを示す |
| 時間軸ラベル | 上端の行 | 週・月・フェーズ等の時間目盛り。バーの位置を読む基準線となる |
| 期間バー | 各レーンのトラック内 | 作業の開始〜終了。位置と長さでタイミングと所要期間を表す |
| マイルストーン記号 | トラック内の特定時点 | 意思決定・納品・完了などの節目を◆等の記号で示す |

## Usage Guide（AIへの使い方）

> 「SLIDE-PATTERN-swimlane-scheduleのレイアウトで、縦のレーンに［ワークストリーム1〜N（担当・作業区分）］、横の時間軸に［週／月／フェーズの目盛り］を置き、各レーンに作業期間のバーと節目のマイルストーン記号を配置してください。デザインはスライドマスターに従ってください。」

**注意点：**
- レーン数は4〜6本、時間目盛りは4〜6区切りが読みやすい上限。これを超える場合は集約するか期間を分割する。
- バーの開始・終了位置は時間軸の目盛りと視覚的に対応させる。目盛りと無関係な位置に置くと期間が読めない。
- マイルストーンは重要な節目のみに絞る。全作業の完了点に置くと記号が氾濫して意味が薄れる。
