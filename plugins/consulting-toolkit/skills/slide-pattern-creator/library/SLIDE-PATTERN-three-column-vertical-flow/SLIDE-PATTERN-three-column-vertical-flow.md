# SLIDE-PATTERN-three-column-vertical-flow

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスターと組み合わせてAIツールに渡すことで、このパターンのスライドを生成できます。タイトルエリア・ページ番号・装飾はスライドマスター（branded-pptx / html-artifactテーマ）で定義されるため、このファイルには含みません。プレビュー .html は生成当時の見本であり非規範。見た目は選択マスターが決めます。

## Overview

**パターン名：** three-column-vertical-flow
**概要：** コンテンツエリアを3カラムに分割し、各カラムに「データソース」「プロセス（番号付きステップ）」「アウトプット」の縦フローを上から下に配置するオペレーティングモデル型レイアウト。カラム間を細い矢印で接続することで工程間の受け渡しを表現する。
**適したシーン：** 3つの機能・チーム・段階が並列に走りつつ、次の段階に成果を渡していく業務フロー。オペレーティングモデル、業務分担、ワークフロー全体の説明に最適。

## Structure（構造）

コンテンツエリアを縦3等分。各カラムは上から「機能名見出し（H3）」「データソースアイコン＋ラベル」「主担当＋番号付きプロセスリスト」「下向き矢印」「アウトプット箱」の順で縦積みされる。カラム間を細い右矢印で接続する。

```yaml
layout: three-column-vertical-flow
zones:
  - role: column
    content: カラム ×3（縦フロー、横並び）
    elements:
      - role: heading
        content: 機能名見出し（H3、アクセントカラー）
      - role: data-source
        content: データソース（アイコン＋ラベル）
      - role: process
        content: 主担当ラベル＋番号付きプロセスステップ 4〜5個
      - role: arrow-down
        content: 下向き矢印
      - role: output
        content: アウトプット箱（ドキュメントアイコン＋成果物名＋頻度）
  - role: inter-column-arrow
    content: カラム間の右向き矢印（前カラムの成果が次カラムの入力になることを示す、任意）
  - role: annotation
    content: スライド下部の補足注記（スコープ精査等）
reading_order: 各カラムは上から下（見出し→データソース→プロセス→矢印→アウトプット）、カラムは左から右
notes: 3カラムが最適（2カラムはtwo-column-compare、4カラムは窮屈になる）。プロセスステップは4〜5個が上限。カラム間矢印は必須ではなく任意。
```

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| 機能名見出し | 各カラム最上部 | この工程・機能の名称（アクセントカラー） |
| データソースアイコン | 見出し直下 | 円柱形（DB）アイコン＋データ源のラベル |
| データソースラベル | データソースアイコン下 | 具体的なデータ源を列挙 |
| 主担当ラベル | プロセスセクション上 | 「主担当：〇〇チーム」形式で担当明示 |
| プロセスステップ | プロセスセクション | 番号付きの4〜5ステップ（AIと人の分担明示） |
| 下矢印 | プロセス下・アウトプット上 | 工程完了→成果物への流れを示す |
| アウトプット箱 | 各カラム最下部 | ドキュメントアイコン＋成果物名＋頻度 |
| カラム間矢印 | カラムの右側 | 前カラムの成果が次カラムのインプットになることを示す |

## Usage Guide（AIへの使い方）

このパターンをAIに指示する際のプロンプト例：

> 「SLIDE-PATTERN-three-column-vertical-flowのレイアウトで、3つの機能（インテリジェンス／フォーサイト／アドバイザリー）を並列カラムで配置し、各カラムに「データソース→プロセス4ステップ→アウトプット」の縦フローを作ってください。デザインはスライドマスターに従ってください。」

**注意点：**
- 3カラムが最適。2カラムだと two-column-compare、4カラムだと窮屈になる
- プロセスステップは4〜5個が上限。ステップ内でAIと人の分担を明示することで生成AI活用の具体性が伝わる
- カラム間の矢印は各カラム完了後の受け渡しを示すため、必須ではなく任意
