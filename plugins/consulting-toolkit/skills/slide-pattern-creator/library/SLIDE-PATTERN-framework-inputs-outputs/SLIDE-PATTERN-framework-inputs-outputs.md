このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスターと組み合わせてAIツールに渡すことで、このパターンのスライドを生成できます。タイトルエリア・ページ番号・装飾はスライドマスター（branded-pptx / html-artifactテーマ）で定義されるため、このファイルには含みません。プレビュー .html は生成当時の見本であり非規範。見た目は選択マスターが決めます。

## Overview

**パターン名：** framework-inputs-outputs
**概要：** スライド中央に大きなフレームワーク（3つのコア機能を横並びで格納した箱）を配置し、左に「インプット」の縦リスト、右に「アウトプット／受け手」の縦リストを矢印で接続する、入力→処理→出力型のフレームワーク図解。
**適したシーン：** 提案するフレームワーク・組織機能・システムの全体像を1枚で示すスライド。インプット源とアウトプット先を明示することで「何が入って何が出るか」を経営陣に伝える場面に最適。

## Structure（構造）

コンテンツエリアを左20%（インプット）・中央60%（フレームワーク中央箱）・右20%（アウトプット／受け手）の3ゾーンに分割。中央の箱は上部にヘッダーバー、中に3つの機能カラムを持つ。中央箱の下または上に共有される「独自データ層」帯を配置することも可能。各ゾーンは矢印で接続する。

```yaml
layout: input-framework-output
zones:
  - role: input
    width: 20%
    content: インプット源のリスト（4項目程度、縦積み）
  - role: framework
    width: 60%
    content: ヘッダーバー＋コア機能カラム ×3（見出し＋説明）＋アウトプット物（各機能の下、対応表示）
    elements:
      - shared_layer: 独自データ層の帯（上部または下部、省略可）
  - role: output
    width: 20%
    content: アウトプット物と受け手のリスト
reading_order: 左（インプット）→ 中央（フレームワーク）→ 右（アウトプット／受け手）
notes: コア機能は3個が視覚バランス的にちょうどよい。独自データ層の帯は省略可（縦2段構成の場合はスペースが取れないため）
```

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| インプット群 | 左カラム | フレームワークに入る情報源を4項目で列挙 |
| 中央フレームワーク箱 | 中央 | 提案する仕組みの全体像。アクセントカラーバーで縁取り |
| コア機能カラム | フレームワーク内 | 3つの機能を横並びで配置（それぞれ見出し＋説明） |
| 独自データ層 | フレームワーク上部帯 | 3機能を通じて蓄積される独自リソースを共通レイヤーで表現 |
| アウトプット物 | フレームワーク下部 | 3機能から生成される成果物（頻度付き）を各機能の直下に配置 |
| 受け手リスト | 右カラム | アウトプットの届け先（経営陣・事業部門 等）を縦に列挙 |
| 接続矢印 | 左右→中央、下→右 | 情報の流れを示すシンプルな矢印 |

## Usage Guide（AIへの使い方）

このパターンをAIに指示する際のプロンプト例：

> 「SLIDE-PATTERN-framework-inputs-outputsのレイアウトで、中央に3つのコア機能を持つフレームワーク箱を配置し、左に4つのインプット源、右に3つの受け手を配置してください。各機能の下にアウトプット物を対応させてください。デザインはスライドマスターに従ってください。」

**注意点：**
- コア機能は3個が視覚バランス的にちょうどよい。2個は薄く、4個以上は窮屈になる
- 中央フレームワークの箱にはヘッダーバー（アクセントカラー塗りつぶし＋白抜き文字）でフレームワーク名を強調する
- 上部の「独自データ層」バナーは省略可（縦2段構成の場合はスペースが取れないため）
