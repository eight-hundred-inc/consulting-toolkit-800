# SLIDE-PATTERN-two-chart-panel
このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。タイトルエリアを除いたコンテンツ領域のみを対象とし、スライド全体のテンプレートではありません。プレビュー .html は生成当時の見本であり非規範。見た目は選択マスターが決めます。

## Overview

コンテンツエリアを左右2つのグラフパネルに分割し、各パネルにタイトルバンド・グラフ本体・凡例を配置するデータ比較レイアウト。2種類のグラフデータを対比・並置して見せる場合に適している。

- **適用場面:** 期間比較、カテゴリ別分析、A/B比較、部門別レポート
- **情報密度:** 中〜高（パネル単位で完結した情報を2セット提供）
- **方向性:** 2つの独立データの並列比較

## Structure

コンテンツエリアを左右2つのグラフパネルに分割し、各パネルにタイトルバンド・グラフ本体・凡例を配置する。左右のパネルは均等幅。

```yaml
layout: two-chart-panel
zones:
  - role: panel
    width: 50%
    content: パネル ×2（横並び、各パネルはヘッダー＋本体で構成）
    elements:
      - role: header
        content: パネルタイトル（見出し帯）
      - role: body
        content: グラフ本体＋凡例（左パネル：ドーナツ1点＋凡例リスト、右パネル：ドーナツ2点横並び）
reading_order: 左パネル → 右パネル、各パネルは上（タイトル）→ 下（グラフ・凡例）
notes: 左パネルのドーナツは4〜5分割、右パネルのドーナツは2〜3分割程度が見やすい。凡例リストは4〜5項目が上限。
```

## Elements

### パネル共通構造
| 要素 | 説明 |
|------|------|
| パネル外枠 | パネルの範囲を示す枠面 |
| ヘッダー | パネルタイトルを表示する帯 |
| タイトル | パネルの内容を示す短い見出し |
| パネル本体 | グラフと付随要素を横に並べて配置するエリア |

### 左パネル（ドーナツ＋凡例リスト）
| 要素 | 説明 |
|------|------|
| ドーナツSVG | 円グラフ1点 |
| 凡例リスト | カテゴリと数値を縦に並べたリスト |
| 凡例ドット | 各カテゴリを識別する色マーカー |
| 各凡例行 | ラベル＋数値（強調） |

### 右パネル（ドーナツ2点横並び）
| 要素 | 説明 |
|------|------|
| ドーナツSVG×2 | 円グラフ2点（横並び） |
| 各グラフラベル | 各ドーナツの内容を示すラベル |
| 配置 | 横並び |

## Usage Guide

- 左右パネルは独立したトピックでもよいが、同じ軸（例：今期vs前期）で対比させると説得力が増す
- 左パネルのドーナツは4〜5分割、右パネルのドーナツは2〜3分割程度が見やすい
- パネルヘッダーのタイトルは10文字以内に収めると見栄えが良い
- 凡例リストは4〜5項目が上限。それ以上はグラフ側でグルーピングを検討する
- パネル間の余白を保つことでパネル独立性を視覚的に示す
