# SLIDE-PATTERN: team-org-with-photo

## 概要

左端にリーダーの丸型写真・名前・役職を配置し、右側にグループ→ユニット→メンバーへと展開する水平ツリー型組織図を配置したレイアウト。

## 適したシーン

チームのマネージャー紹介と組織構造を同時に見せるスライド。採用資料・会社紹介・チーム体制説明。

## Structure

```yaml
layout: two-column
zones:
  - role: leader
    width: 22%
    content: 丸型写真＋名前＋役職
  - role: org_tree
    width: 78%
    layout: horizontal-tree
    content: ユニットノード（＋メンバー数）＋サブレベルノード（＋メンバーアイコン）
reading_order: 左（リーダー）→ 右（組織ツリー：上位ユニット→下位サブレベル）
notes: 接続線がリーダー写真と右ツリーをつなぐ。ユニットは2〜3つが最適（4つ以上は縦が詰まる）。メンバーアイコンは人数分並べ、超過分は省略表記にする。
```

## Elements

| 要素 | 配置 | 役割 |
|---|---|---|
| 丸型写真 | 左カラム中央 | リーダー・マネージャーの顔写真（円形クロップ） |
| 名前 | 写真直下 | リーダーの氏名（大きめ） |
| 役職 | 名前下 | 英語役職名・部門名 |
| 接続線 | 写真と右ツリーを繋ぐ | リーダーと配下の関係を示す水平線 |
| ユニットノード | 右側・上下2行 | チーム・ユニット名（見出しラベル）＋メンバー数 |
| サブレベルノード | ユニットから右に延伸 | ユニットマネージャー・メンバーアイコン群 |
| メンバーアイコン | 最右端 | 人物アイコンで人数を表現（…で省略） |

## Usage Guide

- 丸型写真エリアには必ず円形のクロップ枠を使う
- メンバーアイコンは「人数分並べる＋…（省略）」の表現が効果的
- ユニットが2〜3つの場合に最適。4つ以上は縦が詰まる
- 左バッジは上から下へ配色を変化させるとフローの流れが視覚化される
- ユニットノードの背景色は系統を合わせることで視認性が上がる
