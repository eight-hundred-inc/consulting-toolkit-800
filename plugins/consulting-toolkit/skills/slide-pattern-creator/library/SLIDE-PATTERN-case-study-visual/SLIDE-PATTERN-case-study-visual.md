# SLIDE-PATTERN-case-study-visual

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスター（表紙・タイトル行・フッター・配色・フォント）はマスター側（branded-pptx / html-artifactテーマ等）で定義されるため、このファイルには含みません。プレビュー .html は生成当時の見本であり非規範。見た目は選択マスターが決めます。

## Overview
**パターン名：** case-study-visual
**概要：** 左半分に事例リード（企業属性ラベル＋支援内容2〜3行）、右半分にビジュアル（スクリーンショット・写真コラージュ）のプレースホルダを置く事例紹介。
**適したシーン：** 1件の支援事例・導入事例を見せるとき。実績紹介、活用事例、ケーススタディ。テキストの要点と成果物のビジュアルを対で提示したい場合。
**対応する図解指名：** 事例紹介ビジュアル
**下流実装ヒント：** html-artifact: 2カラム＋画像プレースホルダ（左に属性タグ＋見出し＋支援内容、右に画像） ／ pptx: 2カラム＋画像プレースホルダ

## Structure（構造）
コンテンツエリアを左右半分に分割する。左半分は事例リードとして、企業属性ラベル（業種・規模など）、事例の見出し、支援内容2〜3行、必要なら課題の小ブロックを縦に置く。右半分はビジュアルの大きなプレースホルダ（スクリーンショットや写真コラージュ）にあてる。

```yaml
layout: two-column-case
zones:
  - role: lead
    content: 全幅の要約文（1文・任意）
    position: top-full-width
    optional: true
  - role: case-lead
    content: 事例リード
    position: left
    width: 50%
    elements:
      - attribute-tag   # 企業属性ラベル（業種・規模等）
      - case-heading    # 事例の見出し
      - support-text    # 支援内容2〜3行
  - role: visual
    content: ビジュアル（スクショ・写真コラージュのプレースホルダ）
    position: right
    width: 50%
reading_order: （任意）全幅要約文 → 左（事例リード：属性タグ→見出し→支援内容） → 右（ビジュアル）
notes: 1スライド＝1事例。左テキストは要点のみに絞る。ビジュアルが用意できない場合はプレースホルダのまま残すか、左テキストを全幅に広げて画像枠を「後日差し込み」として明示する
```

## Elements（各要素の役割）
| 要素 | 配置 | 役割 |
|------|------|------|
| 企業属性ラベル | 左・上部 | 業種・規模など事例企業の属性をタグで示す |
| 事例見出し | 左・属性ラベルの下 | 事例を一言で表す見出し |
| 支援内容 | 左・見出しの下 | 何をどう支援したかを2〜3行で記す |
| ビジュアル | 右・半面 | 成果物のスクリーンショットや写真コラージュのプレースホルダ |

## Usage Guide（AIへの使い方）
プロンプト例：「支援事例を1件、左半分に企業属性のタグ＋事例見出し＋支援内容3行、右半分に成果物のスクリーンショットを入れる枠。」

注意点：
- 1スライド＝1事例。複数事例を一覧にしたい場合はカード並列など別パターンにする。
- 左テキストは要点のみ。詳細な数値や課題整理を盛り込みすぎない（右のビジュアルとバランスを取る）。
- ビジュアルが用意できない場合はプレースホルダのまま残すか、左テキストを全幅に広げて画像枠を「後日差し込み」として明示する。
