# SLIDE-PATTERN-two-panel-background-purpose

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスター（表紙・タイトル行・フッター・配色・フォント）はマスター側（branded-pptx / html-artifactテーマ等）で定義されるため、このファイルには含みません。プレビュー .html は生成当時の見本であり非規範。見た目は選択マスターが決めます。

## Overview
**パターン名：** two-panel-background-purpose
**概要：** コンテンツエリアを左右2つのパネルに等分し、各パネルを見出し帯＋箇条書きで構成して、2つの観点（例：背景／目的）を対比・並置する構造。
**適したシーン：** 背景と目的、現状と課題、As-IsとTo-Beなど、対になる2つの観点を並べて示したいスライド。提案書・報告書の導入ページで「なぜ・何を」を1枚で提示する場面に向く。
**対応する図解指名：** 2パネル対比
**下流実装ヒント：** html-artifact: State Grid / 2カラムパネル ／ pptx: 見出し帯付き2パネル

## Structure（構造）
コンテンツエリアを左右2つのパネルに等分（おおむね50:50）する。各パネルは上部に見出し帯（そのパネルの観点名）を置き、その下に箇条書きで内容を列挙する。左右のパネルは同じ構成（見出し帯＋箇条書き）で揃え、見出し帯の高さ・箇条書きの体裁を左右で一致させて対比が成立するようにする。左右で対になる観点（背景／目的、現状／課題 等）を割り当てる。

```yaml
layout: two-panel
zones:
  - role: left-panel
    width: 50%
    content: 見出し帯（観点名）＋箇条書き
  - role: right-panel
    width: 50%
    content: 見出し帯（観点名）＋箇条書き
reading_order: 左 → 右
notes: 左右で対になる観点を割り当てる（背景/目的、現状/課題 等）。見出し帯の高さ・箇条書きの体裁は左右で揃える。
```

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| 見出し帯（左） | 左パネル上部 | 左側の観点名（例：背景）を帯で明示する |
| 箇条書き（左） | 左パネル本体 | 左の観点の内容を箇条書きで列挙する |
| 見出し帯（右） | 右パネル上部 | 右側の観点名（例：目的）を帯で明示する |
| 箇条書き（右） | 右パネル本体 | 右の観点の内容を箇条書きで列挙する |

## Usage Guide（AIへの使い方）

> 「SLIDE-PATTERN-two-panel-background-purposeのレイアウトで、左パネルの見出しを［背景 等］、右パネルの見出しを［目的 等］にし、それぞれの箇条書きに内容を入れてください。左右を対の観点で揃えてください。デザインはスライドマスターに従ってください。」

**注意点：**
- 左右は必ず対になる観点（背景/目的、現状/課題 等）にする。無関係な2項目を並べると対比の意味が生まれない。
- 箇条書きの分量を左右で大きく偏らせない。片方だけ極端に長いと視覚的なバランスと対比が崩れる。
- 観点が3つ以上ある場合はこのパターンでは無理に詰めず、3パネル構成や別パターンを検討する。
