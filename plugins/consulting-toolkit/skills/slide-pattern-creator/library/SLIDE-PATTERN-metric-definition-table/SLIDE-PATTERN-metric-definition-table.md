# SLIDE-PATTERN-metric-definition-table

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスター（表紙・タイトル行・フッター・配色・フォント）はマスター側（branded-pptx / html-artifactテーマ等）で定義されるため、このファイルには含みません。プレビュー .html は生成当時の見本であり非規範。見た目は選択マスターが決めます。

## Overview
**パターン名：** metric-definition-table
**概要：** 調査対象・指標を行に、形式／条件／方式／頻度などの定義属性を列に取り、セルに短句を詰めた大型の定義テーブル。
**適したシーン：** 調査設計や指標体系の定義を1枚で見せるとき。「どの対象を・どの指標で・どんな方式で・どの頻度で測るか」を網羅的に一覧化したい調査方法設計・指標一覧スライド。
**対応する図解指名：** 指標定義テーブル
**下流実装ヒント：** html-artifact: Report Table ／ pptx: ネイティブ表（左端にカテゴリのセル結合列、ヘッダ行に属性名）

## Structure（構造）
コンテンツエリアのほぼ全面を1つの表で使う。左端に大分類（調査対象カテゴリ）のセル結合列を置き、その右に調査対象・指標の行ラベル列、さらに右へ形式／条件／方式／頻度／サンプル数などの属性列を5〜7列並べる。ヘッダ行に属性名、各セルには短句（単語〜1フレーズ）のみを入れ、文章は書かない。

```yaml
layout: full-width-table
zones:
  - role: definition-table
    width: full
    columns:
      - category        # 大分類（セル結合）
      - subject-metric  # 調査対象・指標（行ラベル）
      - attribute-1     # 形式
      - attribute-2     # 条件
      - attribute-3     # 方式
      - attribute-4     # 頻度
      - attribute-5     # サンプル数
    column-count: 5-7
    row-grouping: category-rowspan  # 大分類でセル結合
    cell-style: short-phrase        # セルは短句のみ
reading_order: 上（ヘッダ行）→ 下（各行）、左（大分類・行ラベル）→ 右（属性列）
notes: 大分類でまとめられる対象は左端列をセル結合する。列は5〜7列に抑える。
```

## Elements（各要素の役割）
| 要素 | 配置 | 役割 |
|------|------|------|
| 大分類セル | 左端列（セル結合） | 調査対象を上位カテゴリでグルーピングする |
| 行ラベル | 左から2列目 | 個々の調査対象・指標名を示す |
| ヘッダ行 | 表の最上段 | 各属性列（形式・条件・方式・頻度・サンプル数等）の名称 |
| 属性セル | 本体の各セル | 各対象×各属性の定義値を短句で記す |

## Usage Guide（AIへの使い方）
プロンプト例：「調査対象を法人／個人／従業員の3カテゴリに分け、それぞれの調査指標について『形式・条件・依頼方法・媒体・頻度・サンプル数』を列にした定義テーブルにして。各セルは短い語句だけで。」

注意点：
- 列は5〜7列に抑える。8列を超えると1枚で読めないので、属性を統合するか別スライドに分割する。
- セルは短句厳守。文章になる場合は表ではなく別パターン（比較マトリクス等）を検討する。
- 大分類でまとめられる対象は左端列をセル結合し、視線の起点を作る。分類が不要なら左端列は省略してよい。
