# SLIDE-PATTERN-process-mapping-table

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。スライドマスターと組み合わせてAIツールに渡すことで、このパターンのスライドを生成できます。タイトルエリア・ページ番号・装飾はスライドマスター（branded-pptx / html-artifactテーマ）で定義されるため、このファイルには含みません。

## Overview

**パターン名：** process-mapping-table
**概要：** 業務プロセスを縦のステップとして分解し、各ステップに対して「AIが行うこと」「人間が行う判断」を横列で対応させるマッピングテーブル。左端にステップ番号＋名前、その右にAI役割・人役割の2列、さらに右にAI効果や補足ポイント列を配置する。左サイドに「前半／後半」などの縦グルーピングブラケットを付けて業務の性格を分ける。
**適したシーン：** 業務プロセスへの生成AI導入検討スライド、AI×人間の役割分担の可視化、8〜10ステップ規模の業務工程分析。

## Structure（構造）

コンテンツエリアの左端に細い縦ブラケット（前半／後半）と左端バー、その右に「ステップ列（20%）」「AIエージェント列（30%）」「人間判断列（25%）」「ポイント列（25%）」の4カラム。行数は6〜9ステップ。特定のステップ（AI効果が高い箇所）は行全体に薄い塗りやアクセントを付けて強調する。

    structure:
      layout: process-step-table
      left_grouping:
        style: "縦の中括弧＋ラベル（前半：品質の底上げ／後半：時間の短縮）"
        placement: "テーブル左端"
      columns:
        - name: "ステップ"
          width: 20%
          content: "番号＋ステップ名"
        - name: "AIエージェントが行うこと"
          width: 30%
          content: "AI役割の説明"
        - name: "人間が行う判断"
          width: 25%
          content: "人が担う判断内容"
        - name: "ポイント"
          width: 25%
          content: "各ステップグループの示唆・補足"
      highlight_row: "AI効果高の行に「AI効果高」バッジまたは色付きハイライト"

## Elements（各要素の役割）

| 要素 | 配置 | 役割 |
|---|---|---|
| 縦ブラケット | テーブル最左端 | 業務工程を「前半／後半」など2グループに分ける中括弧 |
| ステップ番号バッジ | ステップ列の左端 | ①②③...の番号アイコン。強調ステップは塗り色を変える |
| ステップ名 | 番号の右 | 「調査票作成」「示唆抽出」など工程名 |
| AI役割セル | 中央左列 | 生成AIが担うタスクを1〜2行で記述 |
| 人役割セル | 中央右列 | 人間が担う判断内容を1〜2行で記述 |
| AI効果高バッジ | 特定行の右端 | 効果の大きさをアクセントカラーで強調 |
| ポイントセル | 右端 | 各グループの示唆・補足（複数行にまたがってもよい） |
| 凡例 | スライド上部または下部 | AI効果の程度を示す凡例（低〜高） |
| フットノート | スライド最下部 | 補足注記（Phase範囲の説明等） |

## Usage Guide（AIへの使い方）

このパターンをAIに指示する際のプロンプト例：

> 「SLIDE-PATTERN-process-mapping-tableのレイアウトで、8ステップの業務プロセスを縦に並べ、各ステップに「AIエージェントが行うこと」「人間が行う判断」の2列を対応させてください。前半と後半で縦ブラケットを分け、AI効果の高いステップにバッジを付けてください。デザインはスライドマスターに従ってください。」

**注意点：**
- ステップ数は6〜9が最適。10以上は1ページに収まらないため分割する
- 縦ブラケットのラベルは「前半／後半」等、業務工程の性格を反映した意味あるものにする
- AIと人の役割は「置き換え」ではなく「協働」の視点で書く（AIが草案、人が最終判断など）
