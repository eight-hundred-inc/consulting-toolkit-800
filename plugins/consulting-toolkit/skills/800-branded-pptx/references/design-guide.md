# 800 Brand Design Guide

> **カラー定数の定義元**: すべてのカラー・フォント定数は [consulting-deck-boilerplate.js](consulting-deck-boilerplate.js) の `C`, `SUB`, `FONT` が唯一の定義元（Single Source of Truth）。本ファイルは仕様の説明のみを行い、HEX値の重複定義はしない。

> **座標の定義元**: すべてのスライド骨格座標は、実PPTXテンプレートのスライドマスター／レイアウトから採寸した値を忠実に反映している。

## カラーパレット概要

### メインカラー（`C` オブジェクト）

| トークン | 用途 |
|---------|------|
| `C.navy` | タイトル・セクション・クロージングの背景。コンテンツスライドのタイトル色 |
| `C.navyLight` | ダーク背景スライドのボトムストライプ |
| `C.accent` | メインアクセント。強調テキスト、マーカー、カテゴリ番号 |
| `C.contentBg` | コンテンツスライド背景（白） |
| `C.text` | 本文テキスト（ダークグレー #404040）。**本スキルの標準テキスト色** |
| `C.textDark` | より濃いテキスト（見出し等で必要な場合） |
| `C.muted` | サブテキスト、フッター、ページ番号、セクションラベル |
| `C.white` | 白 |
| `C.separator` | 見出し下のセパレーターライン（ダークグレー #404040） |
| `C.border` | テーブルボーダー、セクション区切り線 |
| `C.lightGray` | タイトル下セパレーター薄矩形 |
| `C.grayBg` | 薄いグレー背景（タイムライン等） |
| `C.tableBg` | テーブル交互行背景（薄ミント） |
| `C.lightGreenBg` | 薄いグリーン背景（ハイライトエリア） |

### サブカラー（`SUB` オブジェクト）

フレームワーク図、プロセスフロー、カテゴリ区分等で項目を色分けする際に使う。

| トークン | 色名 | 用途例 |
|---------|------|--------|
| `SUB.primary` | グリーン | 1番目の項目、メインカテゴリ、チャートの主系列 |
| `SUB.secondary` | ブルー | 2番目の項目、対比カテゴリ |
| `SUB.tertiary` | イエローグリーン | 3番目の項目、補助。**テキスト色には使わない**（明るすぎる） |
| `SUB.quaternary` | ローズ | 4番目の項目、警告・課題の強調 |

### 背景色の使い分け

| 背景 | 用途 |
|------|------|
| `C.navy`（ダークグリーン） | タイトル、セクション区切り、クロージング |
| `C.contentBg`（白） | 全コンテンツスライド共通 |
| `C.grayBg` | タイムライン・ガントチャートの背景 |
| `C.lightGreenBg` | プロセスフローのヘッダーバー背景 |

## タイポグラフィ

### フォント（`FONT` オブジェクト）

| 用途 | フォント |
|------|---------|
| Latin（英数字） | `FONT.latin` = Century Gothic |
| EA（日本語） | `FONT.ea` = Meiryo UI |
| 見出し | `FONT.head` = Meiryo UI |
| 本文 | `FONT.body` = Meiryo UI |
| 混在テキスト | `splitTextWithFonts()` で自動分割 |

> **重要**: pptxgenjs では `fontFace` に単一フォントしか指定できないため、`splitTextWithFonts()` を使用して英数字に Century Gothic、日本語に Meiryo UI を適用する。

> **`splitTextWithFonts()` と箇条書きの併用禁止**: `splitTextWithFonts()` はテキストを英数字/日本語のセグメントに分割するため、各セグメントに `bullet: true` を付けると1行が複数の箇条書きに分裂する。箇条書きを使う場合は `bulletItem()` ヘルパーを使用する。
> ```js
> // 使用: slide.addText([...bulletItem("項目1"), ...bulletItem("項目2")], { ... });
> ```

### 見出しの色の使い分け

| 見出しの種類 | 色 | 例 |
|---|---|---|
| **主張・メッセージを含む見出し** | `C.accent` | 「コンサルティングのAIトランスフォーメーション」「業務プロセスの再構築が鍵」 |
| **構造ラベル（メッセージなし）** | `C.text`（ダークグレー）, bold | 「背景」「目的」「概要」「ポイント」「設計のポイント」 |

> **ラベル的な見出しをアクセント色にしない**: 「概要」「ポイント」「手法」等の構造的なラベルは情報の区切りを示すだけであり、アクセント色で強調すると過剰になる。アクセント色はメッセージ性のある見出し（そのセクションの主張が伝わるもの）にのみ使う。

### カラムレスポンシブ・フォントサイズ（`FONT_SIZE` テーブル）

カラム数に応じてフォントサイズを自動調整する。`fontSize(category, columns)` 関数で取得する。

| カテゴリ | 1列 | 2列 | 3列 | 4列 | 5列以上 |
|---------|-----|-----|-----|-----|--------|
| `kpi` | 48 | 40 | 36 | 30 | 24 |
| `kpiUnit` | 18 | 16 | 14 | 12 | 11 |
| `title` | 28 | 24 | 22 | 20 | 18 |
| `subtitle` | 20 | 18 | 16 | 14 | 13 |
| `body` | 16 | 14 | 13 | 12 | 11 |
| `caption` | 12 | 11 | 10 | 10 | 9 |
| `badge` | 12 | 11 | 10 | 10 | 9 |

### 固定フォントサイズ（骨格ヘルパーが使用）

| 要素 | サイズ | ウェイト |
|------|--------|----------|
| タイトルスライドのメインタイトル | 36pt | Regular |
| コンテンツスライドのタイトル | 22pt | **Bold** |
| コンテンツスライドのセクションラベル | 12pt | Regular |
| キーメッセージ（タイトル直下） | 12pt | Regular |
| セクションスライドのタイトル | 36pt | Bold |
| セクションスライドの番号 | 16pt | Bold |
| フッター | 8pt | Regular |
| ページ番号 | 12pt | Regular |

## レイアウトパターン仕様

コンテンツスライドの枠（`addContentSlide` が返すスライド）の中で使う10のレイアウトパターン。すべて **コンテンツエリア**（x=0.28, y=1.20, w=9.45, h=3.90）内に配置する。

### 1. 2列分割（Two-Column Split）

左右に分割するレイアウト。中央にアクセント縦線で区切り、または単にスペースで区切る。

| 要素 | 仕様 |
|------|------|
| 左列 | x=`L.contentX`, w=4.5 |
| 右列 | x=5.0, w=4.73 |
| 縦区切り線（任意） | x=4.80, `C.separator`（ダークグレー）, width=1.0 |
| 各列見出し | `C.accent`, bold, `fontSize('subtitle', 2)` |
| 各列本文 | `C.text`, `fontSize('body', 2)` |

**用途**: 背景/目的、概要/ポイント、課題/対応策、左右対比

### 2. データテーブル（Data Table）

行列の構造化データ。ヘッダー行は**太字テキスト＋下線**で区切る（色付き背景は使わない）。

| 要素 | 仕様 |
|------|------|
| テーブル位置 | コンテンツエリア全幅を使用 |
| ヘッダー行 | `C.text`, bold, **背景なし（白）**。下線で区切り |
| データ行（奇数） | `C.contentBg` 背景 |
| データ行（偶数） | `C.tableBg` 背景（任意） |
| ボーダー | `C.border`, width=0.5 |
| テキスト | `C.text`, `fontSize('body', cols)` |

> **テーブルヘッダーに色付き背景を使わない**: サンプルPPTXでは、テーブルヘッダーは太字＋下線のみでシンプルに区切る。色付き背景白文字はAI的な過剰装飾に見えるため禁止。

> **セクションをボックス枠線で囲まない**: KPIカード、マルチパネル、ツリーのルートボックス等、コンテンツ領域を矩形の枠線で囲むのはAI的な過剰装飾。サンプルPPTXでは枠線を使わず、**見出し＋下線**でセクションを区切るか、**薄いグレー背景**（`C.grayBg`）で領域を示す。

> **見出しの下に下線を引く**: 「背景」「目的」「概要」「ポイント」等の構造ラベルの直下に `C.separator`（ダークグレー）の線（0.5pt）を配置し、見出しとコンテンツを視覚的に分離する。→ `addLabel()` ヘルパーを使用。

**用途**: 比較表、評価マトリクス、一覧表、機能比較

### 3. カテゴリ行テーブル（Category Row Table）

左列にカテゴリラベル（アクセント色で強調）、右列に詳細。行間に水平セパレーター。

| 要素 | 仕様 |
|------|------|
| カテゴリラベル（左） | `C.accent` or `SUB.secondary`, bold, w=2.0〜2.5 |
| 詳細（右） | `C.text`, w=残り幅 |
| 行間セパレーター | `C.border`, width=0.5 |

**用途**: セグメント定義、機能別まとめ、論点別整理

### 4. 横棒グラフ＋解説（Horizontal Bar + Commentary）

左にグラフ、右にポイント解説。またはグラフ全幅＋下に解説。

| 要素 | 仕様 |
|------|------|
| バー | `C.accent`（主系列）, `SUB.quaternary`（比較系列） |
| ラベル | `C.text`, `fontSize('body', 2)` |
| 値ラベル | `C.text`, bold |
| 解説エリア | 右側 or 下部、`C.text`, bullets |

**用途**: アンケート結果、ランキング、要因分析

### 5. プロセスフロー（Process Flow）

アクセント背景ヘッダーバー→白ボックス横並び、矢印接続。

| 要素 | 仕様 |
|------|------|
| フェーズヘッダー | `addSectionHeader()` ユーティリティで生成。`C.accent` 背景, 白文字 |
| ステップボックス | `C.contentBg` 背景, `C.border` ボーダー |
| 接続矢印 | `C.muted`, LINE shape |
| フェーズ内テキスト | `C.text`, `fontSize('body', phases)` |

**用途**: 調査の全体像、業務フロー、ロードマップ

### 6. ツリー/階層図（Tree / Hierarchy）

左から右へ分岐するボックス構造。大分類→小分類。

| 要素 | 仕様 |
|------|------|
| ルートボックス | `C.accent` ボーダー or 背景, 太字 |
| 分岐ボックス | `C.border` ボーダー |
| 接続線 | `C.muted`, LINE shapes |
| 末端テキスト | `C.text`, `fontSize('body', 3)` |

**用途**: 論点マップ、バリューチェーン構造、意思決定ツリー

### 7. タイムライン/ガント（Timeline / Gantt）

左にタスク行、上に時間軸。タスクバーとマイルストーン。

| 要素 | 仕様 |
|------|------|
| 時間軸ヘッダー | `C.text`, bold, `fontSize('caption', cols)` |
| タスク行ラベル | `C.text`, bold |
| タスクバー | `C.accent`（主タスク）, `C.grayBg`（サブタスク） |
| マイルストーン | `▲` テキスト, `C.accent` |
| グリッド線 | `C.border`, width=0.5 |

**用途**: プロジェクトスケジュール、フェーズ計画

### 8. コールアウト/KPI（Callout / KPI）

大きな数値 + 小さいラベル。グレー背景ボックスで強調。

| 要素 | 仕様 |
|------|------|
| KPI値 | `C.accent`, bold, `fontSize('kpi', cols)` |
| 単位/ラベル | `C.muted`, `fontSize('kpiUnit', cols)` |
| 背景ボックス（任意） | `C.grayBg`（枠線なし） |

**用途**: 主要指標の提示、ハイライト数値

### 9. セクションサマリ（Section Summary）

見出しラベル（アクセント色テキスト）＋箇条書き。複数セクション縦並び。

| 要素 | 仕様 |
|------|------|
| セクション見出し | `C.accent`, bold, `fontSize('subtitle', 1)` |
| 本文（箇条書き） | `C.text`, bullet=true, `fontSize('body', 1)` |
| セクション間スペース | 0.3" |

**用途**: 章の冒頭サマリ、結論、エグゼクティブサマリ

### 10. マルチパネルグリッド（Multi-Panel Grid）

2x2 or 1x3 のカード状配置。各カードにタイトル＋本文。枠線で囲まず、**見出し＋下線**で区切る。

| 要素 | 仕様 |
|------|------|
| カード区切り | `addLabel()` で見出し＋ダークグレー下線。枠線は使わない |
| カードタイトル | `C.accent`, bold, `fontSize('subtitle', cols)` |
| カード本文 | `C.text`, `fontSize('body', cols)` |
| カード間ギャップ | 0.2" |

**用途**: 複数項目の並列表示、ブランド比較、SWOT的構造

## ユーティリティ関数

### `addSectionLabel(slide, text)`
コンテンツスライド上部の小さなグレーテキスト。章名やカテゴリ名を表示し、スライドの位置づけを示す。`addContentSlide()` の後に呼び出す。

### `addLabel(slide, text, x, y, w, opts?)`
構造ラベル（太字ダークグレー）＋ダークグレー下線を追加。「背景」「目的」「概要」「ポイント」等のセクション区切りに使用。マルチパネルのカード見出しにも使う。opts で fontSize, color 等をオーバーライド可能。

### `addSectionHeader(slide, text, x, y, w)`
アクセント背景＋白テキストのセクションヘッダーバーを追加。プロセスフローのフェーズラベル等に使用。**テーブルヘッダーやセクションラベルには使わない**（それらには `addLabel` を使う）。

### `addCategoryMarker(slide, label, x, y)`
アクセント色丸囲みのカテゴリマーカーを追加。番号やローマ数字の表示に使用。

### `addTwoColumnDivider(slide, y, h)`
アクセント色縦区切り線を追加。2列分割レイアウトで使用。

### `fontSize(category, columns)`
カラム数に応じたフォントサイズを返す。カテゴリ: `kpi`, `kpiUnit`, `title`, `subtitle`, `body`, `caption`, `badge`。

### `bulletItem(text, opts?)`
`splitTextWithFonts()` と `bullet: true` を安全に併用するヘルパー。箇条書き項目を構築する。`slide.addText([...bulletItem("項目1"), ...bulletItem("項目2")], { ... })` のように使用する。
