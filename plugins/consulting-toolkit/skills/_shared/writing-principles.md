# 共通ライティング原則

クライアント向け文書（提案書、最終報告書等）に共通して適用する品質原則。各スキルの固有ルールはスキル側に定義する。

---

## 原則1: 読み手はクライアントである

文書の読み手はクライアント自身であることを常に意識する。

- 「クライアント」「先方」「お客様」→ 社名または「貴社」で記載する
- 自社の作業プロセス（提案書の仮説設計、キックオフでの議論等）への言及は削除する
- 冒頭説明文も「仮説の検証結果と結論を示す」等ではなく「分析結果と示唆を示す」とする

## 原則2: 具体性と定量化を徹底する

曖昧な定性表現を排し、可能な限り具体的な数字・固有名詞・時間軸で記述する。

- ✕ 「市場は拡大傾向にある」「多くの企業が導入している」「コストが高い」
- ○ 「市場規模は2023年の約XX億円から年率X%で成長し、2027年にはXX億円に達する見通し」
- ○ 「導入企業はA社・B社・C社を含む約XX社に上る」

定量データが入手できない場合は「推定XX〜XX」「業界関係者によればおおむねXX規模」等のレンジ表記や定性的な根拠の明示で代替し、数字なしの印象論は避ける。

## 原則3: So What? を常に問う

- 「把握する」「整理する」「深掘りする」だけで終わる文は禁止
- メッセージ行は結論・打ち出しを書く（「把握する/整理する」ではなく「統合して絞る/突破口を提示」）
- 読み手がすでに知っている情報の再整理にとどめない。新たな切り口・示唆を提供する

## 原則4: ファクトの正確性

- 入力情報に存在しない数字・固有名詞・事実を捏造しない
- ソースから引用した数字はそのまま使用し、加工した場合は加工方法を明記する
- 不明な点は「不明」「要確認」と明記し、推定で埋めない
- **すべてのファクト記述には出典URLを添える**。URL がないものは「ファクト」として扱わない（推測・仮説として明示する）
  - 例: 「市場規模 XX 億円（出典: https://…）」
  - 例: 「A 社は XX を導入（出典: 公式プレスリリース URL、要約: 〇〇）」
- 出典 URL は **存在確認をしたうえで** 記載する（404 / リダイレクト先が無関係 / 推測で生成した URL は不可）
- 主要ファクト出典は文書末尾に**台帳形式**で再掲する（脚注番号と URL の対応表）

## 原則5: 読者の前提知識に依存しない

- 略語は初出時に `正式名称（略称）` で展開する（一般ビジネス用語は免除）
- 分類枠組みや専門用語を使う場合、その箇所で最低限の定義を与える
- 結論文には「誰にとっての」示唆かを明記する

## 原則6: 本文は「読み手の意思決定に必要なもの」だけで構成する

クライアント向け文書の本文には、読み手の意思決定を進める論点・分析・示唆だけを書く。作成者側の事情・営業文脈・実施済の経緯・自己言及は、本文から排除する。

**判断基準**: 「この文を消して、読み手の意思決定に必要な情報が欠けるか？」を問う。欠けないなら削る。

### 排除すべきカテゴリ

**作成プロセスのメタ言及**: 書き手がどう構成したか、どんな前提で書いたかは本文に書かない。構成は読み手が読めば分かる。
- ✕ 「与件で示された4テーマ・10項目を、意思決定に直結する『問い』として再構成する」
- ✕ 「事実と仮説を明示的に切り分ける構成原則のもと」

**既往実績・経緯への自己言及**: 過去の作業や経緯は本文に持ち込まない。引用が必要なら根拠として一行添える。
- ✕ 「探索的調査（実施済）の結果を踏まえて」
- ✕ 「Phase 1で分かったことをもとに」（見出しとしての使用も含む）

**営業章・営業フレーズ**: 提案書の本文に売り込み章・自社売り込みフレーズを挿入しない。体制やスケジュールは事実として淡々と記載する。
- ✕ 「中間共有・中間報告」「予算内で品質を担保するための工夫」を独立章として立てる
- ✕ 「貴社が独自に持つ資産を活かして8週間で成果を出す」

**スコープ外の明示的羅列**: 「除外」と書いて長く列挙すると、読み手は「やらないこと」に意識が引っ張られる。スコープ内を書く。本当に紛らわしい1〜2点だけ短く除外明示する。
- ✕ 「スコープ外: 個別ベンダーの詳細PoC実装支援／貴社固有環境への適用設計／…（長い羅列）」

**口語・煽り・AI臭表現**: プロが書く文書として違和感のある表現を避ける。
- ✕ 「そのまま踏襲し」「主に効く論点」「白地図」「シニア」「意思決定に効く」
- ✕ 「立ち上がる」（→ 「合意できた」「定着しつつある」「形になりつつある」など状態語に置換）
- ✕ 「手応えがある」（主体が我々ではなくクライアント。「クライアントから一定の反応があった」など事実記述に）
- ✕ 「中堅ERPベンダーというラベルによる足切りを跳ね返すか？」（煽り）
- ✕ 略号の独断展開・独断省略（「NSM」「L2」など、本文中で初出する略号は与件・社内文書・公的資料に既出かを確認し、ない場合はフル展開で書く）
- ✕ 箇条書きの中に「見出しっぽい一行（体言止め＋鍵括弧）」を入れない（例: 「一次情報の中にも価値の段差がある」のような名詞句行）。箇条書きは「主語＋述語」の完結した一文にする
- ✕ 「クリスタライズして」「凝集して」「圧縮して」「短く」と指示されたのに、同義語の追加で行数を増やす（依頼に反する展開）。原文より短くすることが必須
- ○ 自然なビジネス日本語、与件のニュアンスに沿ったプレーンな表現
