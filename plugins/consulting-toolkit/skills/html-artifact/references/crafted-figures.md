# Crafted Figures — 作り込み図版（Slide Deck format 限定）

8 図解（diagram-components.md）でも 25 コンポーネント（components.md）でも表現しきれない、**そのスライド固有の構造**を、`.fig-NN` 名前空間の per-figure CSS で 1 枚だけ作り込むための文書。Slide Deck format 専用（Vertical Document には使わない）。

これは「新規 CSS を追加しない」原則への**統制された例外**である（components.md / diagram-components.md に明記）。例外が成立する条件は本書の「ガードレール」を満たすこと。

---

## 1. 何が作り込み図版か / いつ使うか

3 層の選択肢の最上位。下から順に検討し、**下で表現できるなら上を使わない**：

1. **情報パネル（components.md 21+4 種）** — リスト・表・カード・KPI 等、情報を整理して見せる
2. **構造化図解（diagram-components.md 8 種）** — フロー / 2x2 / ピラミッド / ファネル / サイクル / ベン / 組織図 / レイヤー。固定 CSS クラスの使い回し
3. **作り込み図版（本書）** — 上記で形にならない、そのスライド専用の構造を per-figure CSS で組む

**使う判断基準**：スライドのメッセージが構造的（マトリクス／時系列ガント／多層構造／対比／関係ネットワーク等）で、かつ 8 図解の既定形に収まらない時のみ。「箇条書きで足りる」「8 図解のどれかで足りる」なら作り込まない。

**典型例**：濃淡で密度を示す 2 軸マトリクス（＋凡例）、マイルストーン付きフェーズ・ガント、縦軸ラベル付きの多層構造、SVG コネクタで結ぶ論点ツリー、左右対比カード（VS 区切り）、評価軸グリッド。— これらは「形の例」であって、毎回これを選ぶメニューではない（§5 参照）。

## 2. fig-canvas / scale 技術仕様

作り込み図版は **大きいネイティブサイズで設計し、`transform: scale` でスライド本文領域に縮小**する。これにより、図版内の文字を 16〜24px 級で余裕をもって組んでも、縮小後にクリスプに収まる。

### スキルの実ジオメトリ（作例の 1488px/0.78 をコピーしない）

template-slides.html の `.slide` は **1280×720・padding 60/64/56** → 本文領域 = **1152×604px**。
Content スライドは上から title-bar（≈70px）＋ message（1 行 ≈50px / 2 行 ≈88px）を消費するため、**図版に使える領域はおよそ 1152 × 440px**（message を 1 行に抑えると ≈460px、図版主役のスライドでは message を短くする）。

### スケール式

```
scale = 1152 / （fig-canvas のネイティブ幅）
制約: ネイティブ高 × scale ≦ 図版領域高（≈440px）
```

- ネイティブ幅は **1300〜1500px** を起点（1152 より広く取り「design big → 縮小」の余裕を作る）。
- 例：ネイティブ幅 1400px → `scale = 1152/1400 = 0.823` → 視覚幅はちょうど 1152px。ネイティブ高は `440 / 0.823 ≈ 535px` 以下に収める。
- 視覚幅を本文幅と完全一致させるため、`scale` は必ず `1152 / ネイティブ幅` で算出する（目分量で 0.8 等と置かない）。

### マークアップとスキャフォルド

template-slides.html に以下のスキャフォルドが定義済み（`.fig-wrap` / `.fig-canvas`）：

```html
<section class="slide" id="sN">
  <div class="title-bar"><span class="sec-num">2.1</span><h2>タイトル</h2></div>
  <p class="message">このスライドの唯一のメッセージ（1 行に抑える）。</p>

  <div class="fig-wrap">
    <div class="fig-canvas fig-01"><!-- ここに作り込み図版の中身 --></div>
  </div>

  <div class="slide-foot">…</div>
</section>
```

per-figure の寸法はデッキの `<style>` に追記する：

```css
.fig-01{ width:1400px; transform:scale(0.823); }   /* scale = 1152/1400 */
.fig-01 … /* 図版内部の独自レイアウト（.fig-01 名前空間に閉じる） */
```

`.fig-wrap{overflow:hidden}` がレイアウト上のはみ出しをクリップする。`transform: scale` は**見た目は縮むがレイアウト箱は縮まない**ため、ネイティブ高を取りすぎると視覚的に収まっていても印刷/サムネイルでクリップする（§6 検証）。

## 3. per-figure scoped CSS 規約

### 名前空間

すべての独自 CSS を **`.fig-NN` 配下にスコープ**する（`.fig-01 .cell{…}` のように）。グローバルな新規クラスを作らない。テンプレート共通 CSS・既存コンポーネント CSS を上書きしない。

### 配色 — テーマ追従を壊さない（必須）

ハードコード色を使わない。配色は次の 2 系統だけから作る：

1. **`--fig-accent`（既定 `var(--accent)`）由来の濃淡ランプ**を `color-mix` で生成：

```css
.fig-01 .shade-1{ background:color-mix(in srgb, var(--fig-accent) 8%,  #fff); }
.fig-01 .shade-2{ background:color-mix(in srgb, var(--fig-accent) 18%, #fff); }
.fig-01 .shade-3{ background:color-mix(in srgb, var(--fig-accent) 30%, #fff); }
.fig-01 .bar    { background:var(--fig-accent); color:#fff; }
.fig-01 .edge   { border-color:color-mix(in srgb, var(--fig-accent) 35%, var(--rule)); }
```

2. **意味色は既存トークン**：良い＝`--good`/`--good-bg`、警告・確認・注意＝`--warn`/`--warn-bg`。新しい意味色トークンを増やさない（作例の「黄色の確認論点帯」も `--warn`/`--warn-bg` で表現する）。

テキスト・罫線は `--ink`/`--ink-soft`/`--ink-mute`/`--rule`/`--rule-soft`、面は `--panel`/`--bg-alt` を使う。

→ これにより作り込み図版は **5 テーマ（Terracotta/Navy/Forest/Charcoal/Mono）すべてに自動追従**する。

### ブランド色のオプトイン（既定では使わない）

特定ブランド色を図版に効かせたいデッキ（例：ビザスク青）では、**そのデッキの `:root` で `--fig-accent` を 1 行上書き**する：

```css
:root{ --fig-accent:#0052FF; }   /* このデッキの作り込み図版だけブランド色になる */
```

シェル（title-bar 等の `--accent`）はテーマのまま、図版内だけブランド色、という二層運用ができる。`.fig-NN` 内に生の `#0052FF` を直書きしない（必ず `--fig-accent` 経由）。

### フォント・形状

- フォントは Noto Sans JP（本文・見出し）／ JetBrains Mono（番号・ラベル・数値）。追加フォント禁止。
- `border-radius` は 4/8/12px（控えめ/標準/強調）。影は使うとしても `0 2px 8px rgba(0,0,0,0.08)` 程度まで。

## 4. 作図文法（grammar）— 主たる指示

**第一の指示は「このスライドの内容に合う図版を構成する」こと。** §5 の作例から選ぶのではなく、次の手順で毎回新しく組む。

1. **メッセージの構造型を見極める** — このスライドが伝える関係は何か：
   - 2 軸の分布／密度 → セル・マトリクス（濃淡で密度）
   - 時間×ワークストリーム → ガント／タイムライン（マイルストーン）
   - 抽象度・時間の層 → 多層スタック（縦/横軸ラベル）
   - 分解・係り受け → ツリー（SVG コネクタ）
   - 対比 → 並置カード（VS／差分強調）
   - 多基準の評価 → 基準グリッド
   - 上記の**組み合わせ**もある（例：フェーズ・ガント＋評価軸）
2. **情報の重みを視覚にマッピングする** — 強調すべき要素に最も濃い `--fig-accent`、補助は淡いランプ、対象外は `--bg-alt`。**強調＝主張の重み**に一致させる（装飾で強調しない）。
3. **読み順を作る** — 左上→右下、軸→セル、上流→下流。視線の動きが論理の流れに一致するよう配置する。
4. **ネイティブサイズで設計** — 1300〜1500px 幅で、文字 16〜24px・余白たっぷりに組む。
5. **scale を算出** — `1152 / ネイティブ幅`。高さが領域（≈440px）に収まるようネイティブ高を調整。
6. **配色を §3 のトークン/ランプだけで塗る。**
7. **§6 で検証**（ライブ・印刷・サムネイル）。

原則（§7 ガードレール）を満たす限り、新しい図版型を発明してよい。

## 5. Worked examples（文法の例示。**メニューではない**）

以下は §4 の文法で組んだ図版の**例**であり、毎回ここから選ぶカタログではない。スライドの内容に合わなければ、これらを使わず新しく組む。構造の要点と配色の当て方のみ示す（全コードは載せない）。

- **濃淡マトリクス＋凡例** — `table` か grid。行＝対象軸、列＝目的軸。各セルを密度に応じ `.shade-1/2/3`（`--fig-accent` ランプ）で塗り、空白セルは破線 `--rule`。凡例で濃淡＝密度を定義。「研究の密集／空白」を一目で見せる時。
- **マイルストーン付きフェーズ・ガント** — 週/月の grid 上にフェーズ帯（`--fig-accent` と淡ランプ）と三角マーカー（節目）。納品物の節目は `--ink` の濃帯で締める。時間×ワークストリームを見せる時。
- **縦軸ラベル付き多層スタック** — 左に縦書き軸ラベル（`writing-mode:vertical-rl`、`--fig-accent`）、右に層カード（上位ほど濃い `--fig-accent`、下位は淡ランプ）。抽象度や時間の層を見せる時。
- **SVG コネクタ付きツリー** — 親（`--fig-accent` ボックス）→ 子（淡ランプ枠）を inline `<svg>` の直線/曲線（`stroke:color-mix(... --fig-accent ...)`）で結ぶ。論点・分解構造を見せる時。
- **VS 区切り対比カード** — 左右カード、中央に「VS」。注目側を `--fig-accent` 枠＋影で持ち上げ、対照側は `--bg-alt`。両側ともライト背景（対称対比でダーク背景にしない＝design-system 禁止パターン）。
- **評価軸グリッド** — 3×2 等のカード、各カードに番号（JetBrains Mono, `--fig-accent`）＋基準名＋説明。左ボーダー `--fig-accent`。選定基準を並べる時。
- **確認論点バー** — 図版内の「要確認」注記は `--warn`/`--warn-bg` の帯で（黄色のハードコードではなく `--warn` 系トークン）。

## 6. 検証（SKILL.md Phase 1 step 11 を拡張）

作り込み図版は per-figure CSS のためクリップ・重なりが起きやすい。step 11 のスクリーンショット確認に次を必ず含める：

- **3 経路で確認**：①ライブ表示 ②印刷（PDF。`@page 1280×720`）③サムネイルパネル。`transform: scale` の入れ子（`.fig-canvas` の scale が `.slide` のビューポート scale の中、さらにサムネイルの `scale(0.125)` の中）で、ライブで収まっても**印刷でクリップ**することがある。印刷は納品 PDF 経路なので必ず見る。
- **クリップ確認**：`.fig-wrap{overflow:hidden}` が「視覚的には収まっているが層ボックスが領域を超える」図版を切る。はみ出すならネイティブ高を下げる（scale を上げる＝ネイティブ幅を縮める）。
- **`@media print` が `.slide` の transform だけを解除し、子孫 `.fig-canvas` の scale を消さない**ことを確認（消すと図版が原寸で溢れる）。

## 7. ガードレール（作り込み＝レイアウトと情報構造であって装飾ではない）

design-system.md の「デザイン禁止パターン」は作り込み図版にも全面適用する：

- 複数アクセント色の禁止 → 配色は `--fig-accent` 由来 + `--good`/`--warn` のみ
- グラデーション・背景画像・テクスチャ・回転/斜め配置・浮遊要素・巨大装飾数字（200px超）・派手なアニメーション → すべて禁止
- 対称的な並列対比でのダーク背景禁止（VS カードは両側ライト）
- 1 スライド 1 図版（複数の作り込み図版を 1 枚に詰めない）
- 8 図解・既存コンポーネントで表現できる構造はそちらを使う（作り込みは最後の手段）

**判断基準**：作り込みの価値は「レイアウトと情報構造の設計」にある。色数を増やす・飾る方向の作り込みはしない。
