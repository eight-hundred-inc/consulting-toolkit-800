# 議事録出力フォーマット詳細ガイド

## 全体構成

議事録は以下の5つのセクションで構成する。

### 1. インタビュー概要

箇条書き形式で基本情報を記載。

```markdown
## インタビュー概要

- 実施日時：2026年1月23日（金）10:00〜10:53
- 所要時間：約53分
- 実施形式：オンライン
- インタビュアー：山田 太郎
```

### 2. インタビュイープロフィール

```markdown
## インタビュイープロフィール

**氏名：山田 太郎（ヤマダ タロウ）**

### 所属・役割
株式会社〇〇 / △△部 / 営業担当

### 専門性
[具体的な専門領域や強みを記載]
```

### 3. ファクトサマリ

インタビュー全体から抽出した重要なポイントを、カテゴリ別に整理。ファクトサマリだけ読む読者がいる前提で設計し、本文にしか書かれていない重要情報がないようにする。

```markdown
## ファクトサマリ

### [カテゴリ名1]
- **[重要ポイント]**：[詳細説明]
- [補足情報]

### [カテゴリ名2]
- **[重要ポイント]**：[詳細説明]
```

**設計上の注意点：**
- 本文の全セクションから示唆を集約する。特定のセクションの情報だけが欠落しないよう注意する
- 同じ数字が異なる文脈で登場する場合（例：臨床試験単体で5億円 vs 承認取得全体で10億円）、ファクトサマリ内で文脈の違いを明記し、混乱を避ける

### 4. インタビュー内容

セクション見出しと質問で構造化する。見出しは `###` までとし、質問は太字テキストで表現する。

```markdown
## インタビュー内容

### [セクション名]

**質問：[質問文をそのまま記載]**

- **[第一階層：核心文]**
    - [詳細1：具体的な説明や事例]
    - [詳細2：背景情報や理由]
    - [詳細3：数値データや実績]
    - [詳細4：課題や注意点]

**サブ質問：[サブ質問文をそのまま記載]**

- **[第一階層：核心文]**
    - [詳細1]
    - [詳細2]
```

**第一階層の書き方**

- 質問への回答の核心を1-2文で表現
- 必ず**太字**で強調
- 完結した文章で、この文だけで回答の要点が理解できること

**良い例：**
```markdown
- **小ロット対応が最大の強みであり、1台から100台程度の製作に対応できる点が競合との差別化ポイントとなっている**
```

**悪い例：**
```markdown
- **小ロット対応が強み**  ← 体言止め、情報量不足
- **小ロットでの対応について**  ← 内容が不明確
```

**第二階層の書き方**

- 第一階層を補足する詳細情報
- 体言止めは使わず、完結した文で記載
- 以下の情報を文字起こしから可能な限り拾い上げる
    - 数値（金額、期間、人数等の定量情報）
    - 具体例（企業名、製品名、技術名等の固有名詞）
    - 判断の理由・背景（なぜそう考えるか、なぜそうなったか）
    - 比較・対比（AとBの違い、過去と現在の変化等）
    - インタビュイー自身の経験に基づく実感・体感
    - 業界の空気感・暗黙知（明言されていないが示唆された構造等）

### 5. まとめ

インタビュー全体から得られた知見を、クライアントの意思決定に資する示唆として総括する。

```markdown
## まとめ

本インタビューを通じて、以下の主要な示唆が得られた。

1. **[ポイント1]**：[説明]
2. **[ポイント2]**：[説明]
3. **[ポイント3]**：[説明]

[総括文]
```

**設計上の注意点：**
- 単なるインタビュー内容の要約ではなく、クライアントの検討文脈（提案書・調査設計書の論点・仮説）と対応づけて示唆を構成する
- インタビュイーの見解がネガティブだった領域についても、どの条件なら成立するかという前向きな論点整理を行う

### 逐語録（オプション）

デフォルトでは含めない。ユーザーから要望があった場合のみ、以下の形式で追加する。

```markdown
## ヒアリングの詳細（主要部分の逐語録）

### [テーマ名]

> **[発言者名]**：「[発言内容をそのまま記載]」

> **[発言者名]**：「[発言内容]」
```

## 文体ルール

### 基本方針

議事録全体を書き言葉（文語体）で統一する。文字起こしは音声認識出力のため口語表現が多いが、そのまま転記せず、書き言葉に変換する。

回答セクションでは、証言の内容そのものを文の主語にする。インタビュー議事録である以上、回答セクションの発言者は自明であり、「誰が言ったか」を毎回フレーミングする必要はない。「〇〇氏は〜と述べた/強調した/指摘した/助言した/見解を示した」等の報告フレームは原則使わない。

### 「」の使用制限

「」は以下の用途に限定し、発言内容の引用には使用しない。

- 固有名詞・制度名（例：「男性型脱毛症診療ガイドライン」）
- 検索ワード（例：「壮年性脱毛症のガイドライン」と検索）

### 主語の変換例（報告フレーム→証言内容が主語）

| 変換前（報告フレーム） | 変換後（証言内容が主語） |
|---|---|
| 横井氏は、今後5年程度では技術の主流は大きく変わらないとの見立てを示した | 今後5年程度では技術の主流は大きく変わらない見通しである |
| 横井氏によれば、事業性の観点から断念されるケースが多かった | 事業性の観点から断念されるケースが多かった |
| 横井氏はこの点を最大の課題として挙げた | この点が最大の課題である |
| 横井氏は営業コンサルの経験から〜と助言した | 営業コンサルの経験を踏まえると、〜が出発点となる |

### 口語→文語の語彙変換例

| 変換前（口語的語彙） | 変換後（文語的語彙） |
|---|---|
| お客さん | 顧客 |
| 営業マン | 営業担当者 |
| 全然違う | 大きく異なる |
| かなり | 相当 / 大幅に |
| やる / やっている | 実施する / 行っている |
| サボる / サボり | 怠慢 / 虚偽報告 |
| うまい人 / ヘタな人 | 巧拙（に個人差がある） |
| ちょっと | やや / 若干 |
| 〜できません | 〜は困難である / 〜に対応できない |
| 〜ってこと / 〜っていう | 〜ということ / 〜という |

### 口語的文の変換例

| 変換前（口語・直接引用） | 変換後（文語） |
|---|---|
| 「5年って言ったら、そこまで変わらないんじゃないかな」 | 今後5年程度では技術の主流は大きく変わらない見通しである |
| 「ビジネスで立たないよねっていう話のほうが結構多かった」 | 事業性の観点から断念されるケースが多かった |
| 「あんまり話を盛り上げなくて申し訳ないんですけど」と前置きしつつ | この領域への参入が厳しいという認識が一貫していた |

### ニュアンスの表現方法

インタビュイーの発言のニュアンス（確信度、温度感、躊躇等）は、報告動詞（「〜と述べた」「〜と強調した」）ではなく、文末表現や副詞で地の文に組み込む。

| ニュアンス | 表現例 |
|---|---|
| 確信を持った判断 | 〜である（断定）、〜は明確に否定された |
| 慎重・懐疑的 | 〜の可能性は低い、〜には慎重な見方が示された |
| 経験に基づく実感 | 実務上は〜である、〜が実態である |
| 強い主張 | 〜が最も重要なポイントである、〜こそが核心である |

### インタビュイーの氏名を主語にしてよい例外

以下のケースに限り、「〇〇氏は〜」の構文を使用してよい。

- デスクリサーチの記述を明確に否定した場合（例：「デスクリサーチでは〜との記述があったが、これは明確に否定された」）
- インタビュイーの個人的経歴・立場が論旨に不可欠な場合（例：プロフィールセクション、経歴の紹介文）
- 複数のインタビュイーの証言を比較・対照する場合（例：「A氏はXと証言したが、B氏はYと証言しており、見解が分かれた」）

## 専門用語の処理

### 初出時の説明

```markdown
什器（じゅうき：店舗で商品を陳列・展示するための器具）
バイヤー（小売店の仕入れ担当者）
リードタイム（発注から納品までの期間）
```

### 略語の展開

```markdown
POC（Proof of Concept：概念実証）
OEM（Original Equipment Manufacturing：相手先ブランド製造）
```

## 質問がスキップされた場合

インタビュー中に回答されなかった質問は、その旨を明記して省略可。

```markdown
**質問：[質問文]**

※ インタビュー中、時間の都合により回答なし
```

または、部分的な回答があった場合：

```markdown
**質問：[質問文]**

- **[得られた情報のみで構成した核心文]**
    - 詳細な回答は得られなかったが、〇〇という言及があった
```
