# workflow.md テンプレート（ポートフォリオ型）

継続クライアントエンゲージメントの初期化時に、以下のテンプレートを使用して `workflow.md` を作成する。

## 適用条件

以下の特徴を持つプロジェクトに適用する。

- 1つの継続クライアント（またはテーマ）の下に、独立したサブプロジェクトが複数並走する
- サブプロジェクトは立ち上げ・完了・アーカイブのライフサイクルを繰り返し、束としての横断管理が必要
- 既存の完了案件・過去案件も含めて棚卸しし、全体の論点・示唆を横断的に維持したい

サブプロジェクトが1つしかない場合や、活動に案件単位の区切りがない月次運用（活動カタログ型が適する）には適用しない。

## テンプレート

プレースホルダ（`[...]` と `{...}`）をヒアリング結果・棚卸し結果で置換して使う。

```markdown
# プロジェクト: [プロジェクト名]（継続クライアントエンゲージメント）

## 基本情報
プロジェクト概要・作業ガイドラインは CLAUDE.md を参照。
- **ワークフロー**: ポートフォリオ管理型（カスタム）
- **ワークフローソース**: custom: workflow.md
- **プロジェクト種類**: 継続クライアントエンゲージメント（複数サブプロジェクト並走）
- **管理開始日**: [YYYY-MM-DD]

## ファイル配置
- CLAUDE.md: プロジェクト概要・作業上の注意・文書作成ガイドライン（静的）
- workflow.md: ポートフォリオ進捗・サブプロジェクト一覧・履歴（本ファイル）
- Output/プロジェクトサマリ.md: 全体の論点・仮説・現時点の示唆
- Output/サブプロジェクト/: 各 active サブプロジェクトの個別サマリ
- {サブプロジェクト名}/: 案件単位の作業ファイル・成果物（サブプロジェクトごとに独立ディレクトリ）
- Input/: クライアント支給資料（編集禁止の場合はその旨を明記）

## 現在の状態
- **フェーズ**: Phase 1（ポートフォリオ初期化）
- **ステップ**: Step 1-1（サブプロジェクト棚卸し） ← 現在
- **ステータス**: not_started
- **担当**: AI

## プロジェクト知識の現状
論点・仮説・示唆は [Output/プロジェクトサマリ.md](Output/プロジェクトサマリ.md) を参照。

## 進捗チェックリスト

### Phase 1: ポートフォリオ初期化（1回のみ）
- [ ] Step 1-1: サブプロジェクト棚卸し ← 現在 [AI]
- [ ] Step 1-2: 全体論点・仮説・現時点の示唆を整理 [AI]
- [ ] Step 1-3: active サブプロジェクトの個別サマリ作成 [AI]

### Phase 2: 継続運用（恒常的）
- [ ] Step 2-1: active サブプロジェクトの進捗トラッキング [AI/人間]（恒常）
- [ ] Step 2-2: 月次レビュー（全 active のステータス棚卸し） [AI]（毎月）
- [ ] Step 2-3: 全体示唆・提供価値の再整理 [AI]（四半期目安）

### Phase 3: 新規サブプロジェクト立ち上げ（都度）
- [ ] Step 3-1: 新規依頼受領・スコープ確認 [人間]
- [ ] Step 3-2: サブプロジェクト立ち上げ（ディレクトリ作成・workflow.md 追記） [AI]
```

## 状態管理の方法

ポートフォリオ層とサブプロジェクト層の2層で管理する。

| 層 | 管理単位 | 語彙 |
|----|---------|------|
| ポートフォリオ層 | 進捗チェックリストのステップ | `not_started` / `in_progress` / `review_pending` / `waiting_human` / `completed` / `skipped` |
| サブプロジェクト層 | サブプロジェクト一覧の行 | `Active` / `Completed` / `Archived` |

運用上のポイント:

- Phase 1 のステップは通常のステップと同様に完了させ、`[x]` を付ける
- Phase 2 のステップは恒常運用のため `[x]` を付けない。実施のたびに履歴へ記録する
- Phase 3 は新規依頼のたびに繰り返す。完了しても `[x]` を付けず、履歴とサブプロジェクト一覧への追加で表現する
- 規模の大きいサブプロジェクトは、そのディレクトリ配下に独自の workflow.md（調査型・チェックリスト型等）を個別に持ってよい。その場合もポートフォリオ側の一覧・個別サマリは維持する
- 個別サマリ（`Output/サブプロジェクト/{名前}.md`）には、そのサブプロジェクトの目的・現状・直近の成果・次のアクションを記載する

## PM 呼び出し時の扱い

PM は Active サブプロジェクト一覧（名前・概要・最終更新）とポートフォリオ層の現在の状態を提示し、AskUserQuestion で以下から選ばせる。

1. 特定の Active サブプロジェクトを進める（個別サマリを読み込んでから作業する）
2. 月次レビュー・全体示唆の再整理を実施する（Phase 2）
3. 新規サブプロジェクトを立ち上げる（Phase 3）

## 更新ルール

1. **サブプロジェクトの状態変化時**: 一覧の該当行を Active → Completed → Archived と移動し、履歴に記録
2. **サブプロジェクト内で作業した時**: 個別サマリの「最終更新」「現状・次のアクション」を更新
3. **新規立ち上げ時**: ディレクトリ作成、Active 表への追加、個別サマリ作成、履歴記録をセットで行う
4. **月次レビュー時**: 全 Active の個別サマリを確認し、停滞案件・完了可能案件を洗い出して履歴に記録
5. **知識状態の変化時**: SKILL.md の自動同期ルールに従い `Output/プロジェクトサマリ.md` を更新
