# メッセージの論理構造（章とスライドの分解）

Phase 2 で章とスライドに分解するときの構造パターン。情報量が1枚に収まらない箇所をどう分割し、メッセージをどう積み上げるかを定める。

---

## メッセージの論理構造型

メッセージの主張には型がある。ボディの構造化やサブメッセージの並び順を決める前に、そのメッセージがどの型かを**先に決める**。

- **消去法型：** 「AでもBでもなく、Cが勝ち筋」「X案・Y案を棄却し、Z案を採る」
- **MECE型：** 複数の軸を漏れなく重複なく列挙（セグメント別・プロセス段階別 等）
- **時系列型：** 過去→現在→未来、Phase1→Phase2→Phase3
- **軸比較型：** 複数対象を同一軸で比較（企業比較、Before/After 等）

### 消去法型の結論位置

消去法型では、サブメッセージ（または章サマリ内の箇条書き）の**並び順を固定する**。

1. A棄却
2. B棄却
3. ...
4. **C（結論）※必ず最後**

結論が先頭や中間に紛れる構造は禁止。消去法の論理は「棄却を積み上げた先に残るもの」を提示することで説得力が出るため、結論を最後に置く順序が読み手の納得形成と一致する。

### サブメッセージの独立性チェック

サブメッセージ群を並べたら、以下の自問を必ず行う。

- 各サブメッセージは、メインメッセージに対して**異なる角度**から支持を与えているか？
- あるサブメッセージを削除したら、メインメッセージの支持力が**明確に弱まる**か？

2つ以上のサブメッセージが同じ観点を異なる表現で言っているだけなら、**1つのサブメッセージ＋複数の証拠**に束ねる。「X証言」「Yガイドライン」「Z市場データ」のように媒体が違っても、同じ事実の別証拠であれば1項目に統合する。

## ピラミッド構造による分割

1枚に収まらない情報量の場合、バーバラ・ミントのピラミッド原則で分割する。

```
サマリスライド（結論・全体像）
├── 詳細スライド1（論拠A）
├── 詳細スライド2（論拠B）
└── 詳細スライド3（論拠C）
```

- サマリスライドのメッセージ = 詳細スライド群の結論を凝縮した1文
- 詳細スライドのメッセージ = サマリを支える個別の論拠や事実
- 読み手がサマリだけ読んでも全体を把握でき、詳細で深掘りできる構造にする
- サマリスライドが不要（詳細が2枚以下 or 結論が自明）な場合は省略してよい

## 問い→回答のカスケード構造

複数のサブトピックを持つセクションで、ピラミッド構造（サマリ→詳細）の代わりに使える。冒頭の「フレーミングスライド」でセクション全体の結論と答えるべき問いを提示し、後続スライドが問いに1つずつ回答する。

```
フレーミングスライド（結論 + 問いの提示）
├── 回答スライド1（問い1への回答）
├── 回答スライド2（問い2への回答）
└── 回答スライド3（問い3への回答）
```

- フレーミングスライドのメッセージ = セクション全体の結論（1文）。ボディに問いの一覧を配置する
- 各回答スライドのメッセージ = 対応する問いへの端的な回答（完全文）
- 問いの数と回答スライドの数は1対1で対応させる。1つの問いに2枚以上使う場合は「問いN続」等で対応を明示する
- 問いは「〜か。」で終わる疑問文で記述する。名詞止めにしない

### ピラミッド構造との使い分け

- **ピラミッド構造が向く場面：** 結論が明確で、根拠を積み上げて説得する場面（分析結果の報告、施策の提言等）
- **問い→回答が向く場面：** 読み手と一緒に考える姿勢を見せたい場面（仮説段階のグランドデザイン、ディスカッション用資料等）

両者を同一資料内で混在させてよい。セクションごとに適した方を選択する。

## カスケードダウン構造（報告書系）

ES（エグゼクティブサマリ）→ 章サマリ → 詳細スライドの3階層で、メッセージを文言レベルで一貫させる。

- 章サマリの第1階層はESの該当箇所と文言を揃える
- 章サマリの第2階層は詳細スライドのメッセージに対応する
- 上位で述べた要素が下位で欠落しない。下位に上位にない要素が唐突に現れない
- サマリスライドは本編を読んでいない読み手にも意味が通るように書く

### ES直後のスライドとの役割分担

ESのメッセージと、ES直後のコンテンツスライド（通常は「背景・目的」）のメッセージは、意図的に役割を分ける。

- **ES：** 「何を提案するか」（提案の結論・全体像）
- **ES直後：** 「なぜ今それが必要か」（構造変化・背景・機会）

両者のメッセージが同一フレーズを共有している場合、直後スライドはストーリーを前に進めていない。並べて読み、「同じことを言っていないか」を確認し、同じなら直後のメッセージを「なぜ」の方向に書き換える。

### 章サマリと詳細スライドの1対N対応

**適用条件：カスケードダウン構造を採用する資料に限る。** 報告書・最終報告書などES→章サマリ→詳細の3階層で論理が積み上がる資料では必須。提案書・ディスカッション資料で「問い→回答」を採用する場合は対象外（問いと回答スライドの1対1対応を代わりに適用する）。

章サマリの各主張は、後続の詳細スライド群と1対N対応を取る。

- 章サマリが主張をM個持つ場合、章内の詳細スライドはM個のグループに分割され、各グループがサマリの対応主張を深掘りする
- サマリに言及がない詳細スライドが存在する場合：(1) サマリに主張として追加する（独立論拠の価値を持つ場合）／(2) 既存主張のサブエビデンスに統合する（補強情報の場合）／(3) 詳細スライドを削除・他章に移動する（本筋から逸脱の場合）

対応関係を読み手に明示するため、章サマリの各主張の末尾に「（→ Slide N）」形式の参照を付記する。複数スライドにまたがる場合は「（→ Slide N-M）」。

### サブメッセージのラベル運用（ナンバリング禁止）

- 章サマリ内では「サブ①」「サブA」等のラベルを付けず、`- **...**` のbulletのみで列挙する
- 詳細スライド側にも「サブA」「サブA-1」等の参照ラベルを付けない。詳細スライドは独立したタイトル＋メッセージで自己完結させる
- 章サマリ→詳細の対応は、章サマリ側の「（→ Slide N）」リンクのみで示す

### 章間の橋渡し

各章サマリには、前章の結論の引き継ぎと次章への接続を1文で明示する。章サマリのボディ冒頭（受けの散文）または示唆の直後に置く。

- 例（前章引き継ぎ）：「本章では第1章で示した『XXが勝ち筋』を受けて、参入先としての供給構造を整理する」
- 例（次章接続）：「残る真の壁への対応は第4章のパートナリング戦略で扱う」

### 章サマリの示唆（So What）

章サマリには、メインメッセージの**行動含意を1文で書く示唆（So What）**を置く。

- 示唆はメインメッセージを受けて取るべき行動を断定で1文
- サブメッセージを単純に要約した抽象化（「4つを踏まえると〜」等）は禁止。メインメッセージの繰り返しになる
- 推奨の語尾：「〜するのが合理的」「〜が必要」「〜を採る」「〜で進める」
- 禁忌の語尾：「〜と考えられる」「〜と思われる」「〜が望ましい」等の弱い断定

メインメッセージが「現状Xである」と診断なら、示唆は「ゆえにYを採るべき」と処方を出す関係性になる。

## ピラミッド構造の整合性検証（Phase 2 で実施）

**適用条件：カスケードダウン構造を採用する資料に限る。** ボディの詳細化に入る前に、以下の階層別通読を必ず実施する。詳細ボディを書いた後の階層修正はコストが大きいため、このステップをスキップしない。

1. **L1抽出：** ES のメッセージを列挙する
2. **L2抽出：** 各章サマリのメッセージを列挙する
3. **L3抽出：** 各章サマリ内の主張（構造化箇条書きのトップレベル）を列挙する
4. **L4抽出：** 各詳細スライドのメッセージを列挙する

次に、各階層の対応関係を検証する。

- L1のN要素 ⇔ L2のN章に過不足なくマップされるか
- L2の章結論 ⇔ その章のL3主張群から演繹可能か
- L3の各主張 ⇔ 相互に独立した角度から章結論を支持しているか
- L3の各主張 ⇔ L4の詳細スライドに1対N対応しているか
- L4のスライドで、上位（L3）にマップされないものはないか

マッピングに齟齬がある場合、構造を再設計してからボディを書く。検証結果はユーザーに提示し、ピラミッド構造の階層表を見せた上で合意を得てから Phase 3 に進む。

提案書・ディスカッション資料で「問い→回答」を採用する場合は、本検証の代わりに問い→回答の1対1対応を検証する（回答スライドが問いの数と一致しているか、各回答が対応する問いに端的に答えているか）。
