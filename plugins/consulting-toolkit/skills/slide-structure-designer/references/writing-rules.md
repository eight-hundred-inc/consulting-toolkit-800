# ライティング規約（ボディを書くとき）

Phase 3 で各スライドのボディを書くときの規約。文の書式・ボディの構成・表現形式・出典の付け方を定める。

表現の禁忌（AI典型表現・比喩名詞/動詞・俗語・戦闘メタファ・弱い断定）と対顧客スタンスは、[quality-checklist.md](quality-checklist.md) の **F. 表現** と **I. 対顧客スタンス** に集約した。書きながらそれを意識し、Phase 4 で点検する。

---

## 共通ライティング原則（全スライドに適用）

- 読み手はクライアント自身。「貴社」で書く。作業プロセスへの言及は削除する
- 曖昧な定性表現ではなく、具体的な数字・固有名詞・時間軸で記述する
- すべての記述が「So What?」に答えている。整理・把握で終わらない
- ソースにない数字・事実を捏造しない。不明は「不明」と明記する
- **ファクト記述には出典を必ず添える**。URLがないものは「ファクト」として扱わず推測・仮説として明示。URLは存在確認したうえで記載（404/推測URL不可）
- 略語は初出展開。分類・専門用語はその場で定義。結論文に主語を明記する
- 本文は「読み手の意思決定に必要なもの」だけで構成する。作成プロセスのメタ言及・既往実績への自己言及・営業フレーズ・スコープ外の長い羅列・口語/煽り/AI臭表現は排除する

---

## メッセージの書式

- メッセージ行は句点（。）で終わる完全な文にする。体言止め禁止
- メッセージ行にダッシュ（—、--）を使わない
- 1つのメッセージ行に2つ以上の論点を入れない。入った場合はスライドを分割する
- 否定形（「AはBではない」）で止めず、「つまりCである」まで書き切る
- メッセージはデータの説明（「〜を分析した」）ではなく発見事実（「〜であることが判明した」）を述べる
- ぼかし言葉（など・等・かもしれない）を消して言い切る。専門語は相手のレベルに合わせて言い換える
- **重要なメッセージ・タイトルは、確定前に複数案を出してから最良を選ぶ**（型：数字／問いかけ／対比／体言止め）。入れられる数字は入れて具体的にする（ただし事実にない数字は作らない）

## ボディの構成

### ボディはメッセージ補強方向のみ

ボディの全要素はメッセージを補強する根拠でなければならない。

- メッセージの主張と矛盾・相殺する事実を同じスライドに併記しない
- 留保・例外を述べる必要がある場合は別スライドで扱う
- 各項目について「これはメッセージを支えているか？」を検証し、支えていない項目は削除または別スライドに移す
- 「診断」（現状分析）と「処方」（施策提言）を1枚に混ぜない

### ボディ内部のストーリーフロー

ボディは次の4要素を基本構造とする。すべてを常に含める必要はないが、「受け」は原則省略しない。

1. **受け（1〜2文）：** メッセージの主張をボディ冒頭で受け止める散文。メッセージからボディへ視線が移ったとき文脈が途切れないようにする
2. **根拠：** メッセージを支える具体的なデータ・事実・論拠。表・構造化箇条書き・図解のいずれかで提示する
3. **展開：** 根拠から導かれる具体的な含意・詳細・適用。根拠だけでは「So What?」に答えていない場合に必要
4. **橋渡し（任意・1文）：** 次スライドの話題を予告する散文。自明な場合は省略する

ボディがいきなりテーブルや箇条書きで始まると、メッセージとの接続が切れる。

### ボディの具体性

- 元資料のテーブルは内容を保全した上で、メッセージの性質に合った表現形式（表・構造化箇条書き・散文）に変換する
- 箇条書き・フロー・ステップは原文の粒度を保つ
- 図解が必要な箇所はテキストで構造を記述し、スライド化時に図に置き換える想定を明記する

## 表現形式：表と散文の使い分け

ボディの表現形式は、ソースの形式ではなくメッセージの性質に従う。

### 表を使う場面

- **比較：** 複数の対象を同じ軸で並べて違いを示す（企業比較、Before/After、段階モデル等）
- **一覧参照：** 読み手が必要な行を探して参照する（成果物リスト、スケジュール、費用等）
- **外部フレームワークの引用：** 定型構造を提示する場合

### 散文＋構造化箇条書きを使う場面

- **主張の展開：** メッセージの主張を根拠とともに論証する
- **プロセスの説明：** 段階的な流れを意味づけを含めて記述する
- **概念の定義：** 新しい概念を導入し理解させる

ソースが表形式でも、メッセージが「比較」でなく「主張」を述べるなら散文に変換する。逆も同様。

### 構造化箇条書きの書式

各項目は「**太字リード：** 説明文」の2層構造にする。太字リードが主題を端的に示し、説明文が「なぜ重要か」を補う。

- 良い例：**醤油浸透市場の構造的差異：** 食文化圏ごとの受容パターン、浸透の段階と条件。海外売上比率8割超の実績に裏付けられ、他社には蓄積できない知見である
- 悪い例（フラットな羅列）：醤油浸透市場の構造的差異 / 食文化圏ごとの受容パターン / 浸透の段階と条件

文末表現は全項目で統一する（動詞止めなら全項目を動詞止めに）。

## 情報量の保全

スライド構成MDは単体で報告書として成立する情報量を優先する。

- 紙面の制約はスライド制作（PPTX生成）時に対処するもの。構成段階では情報量を落とさない
- ソースの数値・事実・根拠を省略せず転記し、プレゼンターが理解に困らない詳細度を確保する

### 表現形式の変換は省略ではない

情報量の保全は内容（数字・事実・根拠）に適用する。表現形式（表・箇条書き・散文）はメッセージの論理構造に合わせて変換してよい。以下は省略に該当しない。

- ソースの表を、メッセージの性質に合わせて構造化箇条書きに変換する
- 詳細度の高い情報を本編からAppendixに移し、本編ではメッセージの抽象度に合った粒度で提示する
- ソースの箇条書きを、文脈を補う散文に再構成する

「情報量の保全」を根拠にソースの表をそのままボディに貼り付けるのは、このルールの誤用。

---

## スライドの単体完結性と前後の接続

- 図・データとそれに言及するテキストは必ず同一スライド内に配置する
- スライド構成MD内で他スライドの番号をハードコード参照しない（「Slide 12参照」等）。組み換え時に参照が壊れる
- **サマリスライド・章扉・章間遷移**：散文的な橋渡し（「次に〜を確認する」「前スライドで定義した〜を踏まえ」）を許可する。ストーリーの連続性のために推奨
- **章内の詳細スライド**：単独完結性を優先し、後続予告の橋渡し文（「〜は次のスライドで示す」「後述」「Day N への接続」）は原則削除する。橋渡しがないと意味不明になる構成自体を見直す
- 橋渡し文を置く場合はボディ末尾に1文。中盤に他スライド前提の記述を混ぜない

### 前スライドとの重複排除

- 各スライドは新しい情報に焦点を当てる。前スライドで完結した要素は繰り返さない
- スライドを分割した場合、前スライドで述べた要素を後スライドから削除する

---

## ソースの内容を変えない

ソースの内容（数字・固有名詞・主張・根拠）は変更しない。構成の組み替え・統合・分割はするが、情報の追加・削除・言い換えはしない。ユーザーが明示的に「仮説は不要」「根拠は省略」等と指示した場合のみ、該当部分を削除する。

---

## 図表メタ情報の必須記載

すべての図表に以下のメタ情報をMarkdownテキストとして付記する（画像ファイル内には埋め込まない）。

- 図の見出し（キャプション）／データソース（設問番号＋設問テキスト）／対象者条件／N数

フォーマットは全図で統一する。例：`図X: [見出し]（[設問番号]: [設問テキスト]、対象: [条件], N=[数]）`

---

## 出典の記載

具体的な数値・事実・フレームワーク・事例に出典を付記する。公知の事実（「AIの普及が進んでいる」等）には不要。

### 本文内マーカー方式

本文内で出典を参照する箇所に `※N`（全角※＋半角数字）のマーカーを付ける。スライド末尾にマーカー番号と対応する出典を一覧で記載する。

```
...30種類以上のOTC製品が乱立している。※1
...が確認された。※1※2
```

```
**出典：**

※1: [出典情報]
※2: [出典情報]
```

### 出典タイプ別フォーマット

| タイプ | フォーマット |
|---|---|
| 外部文献・レポート | `※N: [著者/発行者] "[タイトル]" ([年]) [URL]` |
| 記事・Webサイト | `※N: [媒体名] "[タイトル]" ([年月日]) [URL]` |
| エキスパートインタビュー | `※N: エキスパートインタビュー（肩書き＋役割。例: 元研究者）` ※氏名・固有名・日付は記載しない |
| 社内調査・クライアント提供情報 | `※N: [クライアント名]社内調査（調査内容、年）` |
| デスクリサーチ成果物 | `※N: [展開された公開URL／一次情報源]`（社内符号 `DR-XX` のままにしない。上流の `integrated-analysis-creator` で展開済みのソースを記載する） |

表内の出典は、該当セルに `※N` を記入し、実際の出典名は末尾の出典リストで統合管理する。

### スライド本文内のインタビュー引用

スライド本文中でインタビュー知見を引用する場合、その箇所には**出典マーカー（※N）を付けない**。

- 引用元は本文中で肩書き＋役割を明示することで足りる（例：「元研究者によれば〜」）
- **出典：** セクションには、エキスパートインタビュー全体の対象者構成（例：「元研究者3名、元営業1名」）のみ記載する形でも可
- 固有名（氏名）はスライド本文・出典リストいずれにも記載しない

### 採番ルール

- スライド内登場順に1から採番する（スライドをまたいだ連続採番はしない）
- 1スライド内で同じソースを再引用する場合は同じ番号を再利用する
- 論考主体で出典が不要なスライドには **出典：** セクションを付与しない
- ソースにURLがある場合はそのまま転記する。ない場合は「※URL要確認」と注記し、可能なら補完する
