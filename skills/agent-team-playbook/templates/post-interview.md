# インタビュー後: Agent Team テンプレート

複数のインタビュー文字起こしが溜まった段階で、議事録の作成と追加デスクリサーチを並行で実行する。

**注意**: 議事録は1件ずつ記憶が新しいうちに逐次処理するのが実務の主流。このテンプレートは文字起こしが複数件溜まった場合のバッチ処理用。1件だけ処理する場合は interview-minutes-creator スキルを直接使うほうが適切。

## 前提条件

- `Input/インタビュー/` に文字起こしファイルが複数配置済みであること
- `Output/提案書.md` が存在し、論点・仮説が記載されていること
- `Output/インタビューガイド.md` が存在すること

## チーム構成

- **Lead（リードエージェント）**: 全体の調整とインタビューまとめを担当
- **Transcriber（1-2名）**: 文字起こしから議事録を作成。ファイル数に応じて人数を調整
- **Researcher**: desk-researcher SubAgent を使って追加デスクリサーチを並行実行

## team-prompt パターン

```
インタビュー完了後の一括処理を Agent Team で実行してください。

## 入力
- 文字起こし: {Input/インタビュー/ 内のファイル一覧}
- インタビューガイド: Output/インタビューガイド.md
- 提案書: Output/提案書.md

## チーム構成
Input/インタビュー/ 内のファイル数を確認し、以下のようにチームを構成すること。

- Transcriber-A: 以下の文字起こしから議事録を作成する。
  interview-minutes-creator スキルを参照し、Output/議事録/ に出力すること。
  担当ファイル: {ファイルリストの前半}

- Transcriber-B（4件以上の場合）: 以下の文字起こしから議事録を作成する。
  interview-minutes-creator スキルを参照し、Output/議事録/ に出力すること。
  担当ファイル: {ファイルリストの後半}

- Researcher: desk-researcher SubAgent を使って追加デスクリサーチを実行する。
  Output/提案書.md の論点を参照し、インタビューで判明したギャップを埋める調査を行う。
  Output/調査/追加調査.md に出力すること。

- Lead（あなた）: Transcriber と Researcher の完了を待ち、以下を実行する。
  1. 全議事録を読み込み、論点ごとに示唆を抽出してインタビューまとめを作成する
  2. Output/インタビューまとめ.md に出力する
  3. 全成果物のパスと要約を提示し、レビューを待つ

## ルール
- 各 Transcriber は異なるファイルを担当する（ファイル競合を防ぐ）
- Researcher に plan approval を要求すること
- スキルを使う際は該当する SKILL.md を読み込んでから実行すること
```

## 成果物

| 成果物 | 格納先 | 担当 |
|--------|--------|------|
| 議事録（各件） | Output/議事録/{対象者名}_議事録.md | Transcriber |
| 追加調査結果 | Output/調査/追加調査.md | Researcher |
| インタビューまとめ | Output/インタビューまとめ.md | Lead |

## カスタマイズのポイント

- `{ファイルリスト}`: Input/インタビュー/ 内の実際のファイルパスに置換する
- ファイル数が3件以下なら Transcriber は1名で十分
- 追加デスクリサーチの焦点はインタビューで得られた新しい論点・ギャップに合わせて調整する
- 議事録のフォーマットはインタビューガイドの質問構成に沿う
