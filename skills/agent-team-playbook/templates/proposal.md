# 提案フェーズ: Agent Team テンプレート

与件を受領した直後に、提案書の作成と初期デスクリサーチを並行で実行する。

## 前提条件

- `Input/与件.md`（またはクライアント提供の与件資料）が配置済みであること
- CLAUDE.md が存在し、プロジェクト概要が記載されていること

## チーム構成

- **Lead（リードエージェント）**: 提案書の作成を担当
- **Researcher**: desk-researcher SubAgent を使ってデスクリサーチを並行実行

## team-prompt パターン

以下のパターンをプロジェクトの状態に合わせてカスタマイズし、team-prompt として使用する。

```
このプロジェクトの提案一式を Agent Team で作成してください。

## 入力
- 与件: {与件ファイルパス}

## チーム構成
- Researcher: desk-researcher SubAgent を使って初期デスクリサーチを実行する。
  調査結果を Output/調査/初期調査.md に出力すること。
  提案書の論点・仮説をもとに、市場動向・技術動向・競合情報を収集する。

- Lead（あなた）: 提案書を作成する。
  interview-research-proposal スキルを参照し、Output/提案書.md に出力すること。

## ワークフロー
1. Lead は与件を読み込み、提案書ドラフトの作成を開始する
2. 同時に Researcher を spawn してデスクリサーチを並列実行する
3. 両方が完了したら、調査結果を踏まえて提案書を更新する
4. インタビューガイドを作成する（Output/インタビューガイド.md）
   interview-guide-creator スキルを参照すること
5. 報告書骨子を作成する（Output/報告書骨子.md）
   report-outline-creator スキルを参照すること
6. 完了後、全成果物のパスと要約を提示し、レビューを待つ

## ルール
- Researcher に plan approval を要求すること
- 各ファイルは担当者だけが編集する（ファイル競合を防ぐ）
- スキルを使う際は該当する SKILL.md を読み込んでから実行すること
```

## 成果物

| 成果物 | 格納先 | 担当 |
|--------|--------|------|
| 提案書 | Output/提案書.md | Lead |
| 初期調査結果 | Output/調査/初期調査.md | Researcher |
| インタビューガイド | Output/インタビューガイド.md | Lead |
| 報告書骨子 | Output/報告書骨子.md | Lead |

## カスタマイズのポイント

- `{与件ファイルパス}`: 実際のファイルパスに置換する。docx の場合は docx-to-markdown-with-references スキルでの変換指示を追加する
- プロジェクト種類に応じてリサーチの焦点を追記する（技術動向なら技術課題・参入障壁、市場調査なら市場規模・セグメント等）
- クライアントから追加の参考資料が提供されている場合は入力に追加する
