# Agent Team Playbook

Claude Code の Agent Team 機能を活用して、コンサルプロジェクトの並行処理を実行するためのテンプレート集。

## Agent Team とは

Claude Code の Agent Team は、リードエージェントが複数のチームメイトを動的に生成し、独立したタスクを並列実行する機能。各チームメイトは独立したコンテキストで動作し、共有タスクリストとメッセージでリードと連携する。

## いつ使うか

以下の条件を満たす場面で Agent Team が有効:

- 同時並列で回せる独立タスクが **3つ以上** ある
- 各タスクが **異なるファイルを出力** する（競合しない）
- 1セッション内で **完結** できる規模
- 逐次実行と比べて **時間短縮の実益** がある

これを満たさない場面では、通常の逐次ワークフロー（project-manager + 個別スキル）のほうがシンプルで品質管理しやすい。

## 使い分け

| 場面 | 推奨 | 理由 |
|------|------|------|
| 与件受領 → 提案一式 | Agent Team | 提案書と調査を並列で作成でき、成果物が独立している |
| クライアントフィードバック反映 | 逐次 | 単一ファイル修正。並列不要 |
| インタビュー1件の議事録 | 逐次 | 単一タスク。チームはオーバーヘッドになる |
| 文字起こしが複数溜まった場合 | Agent Team | 複数議事録 + 調査を並列で処理。ファイル数が多いほど効果が出る |
| 分析の方向性検討 | 逐次 | 人間との対話的探索が必要 |
| 報告書ドラフト + レビュー | Agent Team | 多角レビューの並列実行 |
| スライド作成 | 逐次 | 800-branded-pptx スキルで逐次処理 |
| 個別の修正・ブラッシュアップ | 逐次 | ファイル追加 → スキル実行 → レビューの反復 |

## テンプレート一覧

| テンプレート | ファイル | 起動タイミング | 並行するタスク |
|-------------|----------|---------------|---------------|
| 提案 | [templates/proposal.md](templates/proposal.md) | 与件受領後 | 提案書ドラフト + 初期デスクリサーチ |
| インタビュー後 | [templates/post-interview.md](templates/post-interview.md) | 文字起こしが複数溜まった場合 | 議事録作成(複数) + 追加デスクリサーチ |
| 報告書 | [templates/report.md](templates/report.md) | 分析方向性が固まった段階 | 報告書ドラフト → ファクトチェック + 反論レビュー |

各テンプレートは team-prompt のパターンであり、実行時にAIがプロジェクト状態（実在するファイルパス、進捗、論点）で変数を埋めて使う。

## 実行方法

1. Claude Code でプロジェクトフォルダを開く（CLAUDE.md が自動ロードされる）
2. 自然言語で指示する:
   - 「Agent Teamで提案を作成して」
   - 「インタビュー結果をAgent Teamで並行処理して」
   - 「報告書をAgent Teamで作成して」
3. AIがこのスキルの該当テンプレートを読み込み、プロジェクトの現在の状態に合わせてカスタマイズした team-prompt を生成・実行する

## テンプレートのカスタマイズ

テンプレート内の `{変数}` は実行時に以下の情報で置換する:

- プロジェクトフォルダ内の実在ファイルパス（Input/, Output/ の中身を確認）
- workflow.md の進捗状態
- 提案書・調査結果等から抽出した論点・仮説

## インタビュー後テンプレートについて

議事録は1件ずつ、記憶が新しいうちに逐次処理するのが実務の主流。このテンプレートは文字起こしが複数件溜まった場合のバッチ処理用として用意している。1件だけ処理する場合は interview-minutes-creator スキルを直接使うほうが適切。

## 注意事項

- Agent Team は Claude Code 固有の機能であり、Cursor では利用できない
- Cursor で逐次ワークフローを進める場合は project-manager スキル + pm-start コマンドを使う
- テンプレートはプロジェクトフォルダにコピーしない。このスキルディレクトリに保管し、実行時に参照する
