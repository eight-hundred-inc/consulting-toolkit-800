# Project Manager

プロジェクトの進行を管理する汎用オーケストレーター。プロジェクト種類に応じたワークフロースキルを呼び出し、状態管理と進捗追跡を行う。

## 対応ワークフロー

| プロジェクト種類 | ワークフロースキル | 説明 |
|------------------|-------------------|------|
| **調査プロジェクト** | `research-project-workflow` | 技術動向調査、市場調査、インタビュー中心、DD |
| コンテンツ作成 | （将来追加） | 資料・レポート作成 |
| 事業計画・戦略策定 | （将来追加） | 事業計画、戦略立案 |
| ソフトウェア開発 | （将来追加） | 開発プロジェクト |

## トリガーと処理

### 1. プロジェクト初期化

**トリガー**: 「プロジェクトを開始」「新規プロジェクト」「ワークフロー開始」

**処理手順**:
1. AskQuestionツールでプロジェクト種類を確認
2. 対応するワークフロースキルを特定
3. プロジェクト情報をヒアリング:
   - プロジェクト名
   - クライアント名（あれば）
   - 納期
   - 予算（あれば）
   - スコープ（1-2行）
4. `CLAUDE.md`を作成（テンプレート: [references/claude-md-template.md](references/claude-md-template.md)）
   - ヒアリング結果で変数を置換する
   - フォルダ構造（`Input/`, `Output/`, `Output/議事録/`, `Output/調査/`）も同時に作成
5. `workflow_status.md`を作成（テンプレート: [references/status-template.md](references/status-template.md)）
6. **ワークフロースキルを読み込む**
7. 入力ファイルの有無を確認
8. Step 1開始の確認

### 2. 進捗確認

**トリガー**: 「進捗確認」「現在の状況」「どこまで進んだ」

**処理手順**:
1. `workflow_status.md`を読み込み
2. ワークフロースキルを読み込み（ステップ情報取得）
3. 現在のフェーズ・ステップを報告
4. 完了済みステップと残りステップを一覧表示
5. 次のアクションを提案

### 3. ワークフロー継続

**トリガー**: 「次のタスク」「続きを進めて」「次のステップ」

**処理手順**:
1. `workflow_status.md`を読み込み
2. **ワークフロースキルを読み込み**（現在のステップの詳細を取得）
3. 現在のステップの担当を確認:

   **AIタスクの場合**:
   - ワークフロースキルの指示に従って実行
   - 必要に応じて連携スキルを呼び出す
   - 完了後、`workflow_status.md`のステータスを `review_pending` に更新
   - **レビューゲートへ進む**（後述）

   **人間タスクの場合**:
   - ワークフロースキルの指示に従って案内
   - 「○○が完了したら教えてください」と待機

### 4. レビューゲート

AIタスク完了後、次のステップに進む前に必ずユーザーのレビューを受ける。

**処理手順**:
1. **成果物の要約を提示**:
   - 出力ファイルのパス
   - 主要な内容・変更点の要約（3-5行）
2. **品質チェックを実行**:
   - `quality-reviewer` SubAgentに成果物と品質チェック項目を渡す
   - SubAgentが返却した品質チェック結果を報告
3. **AskQuestionツールで判断を仰ぐ**:
   - **承認**: `workflow_status.md`のステータスを `completed` に更新し、次のステップを案内
   - **修正指示**: ユーザーからフィードバックを受領し、該当箇所を修正して再度レビューゲートへ
   - **スキップ**: `workflow_status.md`のステータスを `skipped` に更新し、次のステップへ進む

**レビューゲートのAskQuestion選択肢**:
```
質問: このステップの成果物を確認してください。
選択肢:
  - 承認（次のステップへ進む）
  - 修正指示がある（フィードバックを伝える）
  - スキップ（このステップを飛ばして次へ）
```

### 5. レビュー承認

**トリガー**: `/pm-approve`、「承認」「OKです」「次に進んで」

**処理手順**:
1. `workflow_status.md`で `review_pending` 状態のステップを特定
2. ステータスを `completed` に更新
3. チェックリストを `[x]` に更新、次のステップに `← 現在` を移動
4. 成果物リンクを更新
5. 履歴に追記
6. 次のステップを案内

### 6. 人間タスク完了報告

**トリガー**: 「○○が完了」「○○しました」「○○終わりました」

**処理手順**:
1. 完了したタスクを特定
2. 成果物の格納場所を確認（必要に応じて）
3. `workflow_status.md`を更新
4. 次のステップ（AIタスク）の開始を提案

### 7. Agent Team による並行処理

**トリガー**: 「Agent Teamで〜」「並行処理で〜」「チームで〜」

**処理手順**:
1. `workflow_status.md`を読み込み、現在のフェーズ・進捗を確認
2. `agent-team-playbook` スキル（`skills/agent-team-playbook/SKILL.md`）を読み込む
3. 現在のフェーズに適したテンプレートを選択:
   - Phase 0（提案前）: `templates/proposal.md`
   - Phase 1（インタビュー後、文字起こしが複数ある場合）: `templates/post-interview.md`
   - Phase 2（分析方向性確定後）: `templates/report.md`
4. テンプレート内の `{変数}` をプロジェクトの現在の状態で置換:
   - 実在するファイルパス（Input/, Output/ の中身を確認）
   - 提案書・調査結果等から抽出した論点・仮説
5. カスタマイズした team-prompt を生成し、Agent Team を実行する

**注意**: Agent Team は Claude Code 固有の機能。Cursor では利用できないため、逐次ワークフローで対応する。

## 状態管理

`workflow_status.md`で進捗を管理。フォーマットは [references/status-template.md](references/status-template.md) を参照。

**重要**: 各ステップ完了後、必ず`workflow_status.md`を更新すること。

## ワークフロースキル呼び出しのルール

1. **プロジェクト初期化時**: 対応するワークフロースキルを特定し、SKILL.mdを読み込む
2. **ステップ実行時**: ワークフロースキルの指示に従い、必要な連携スキルを呼び出す
3. **スキル読み込み順序**:
   ```
   1. project-manager（このスキル）
   2. 対応するワークフロースキル（例: research-project-workflow）
   3. ステップで必要な連携スキル（例: interview-research-proposal）
   ```

## ワークフロースキルの呼び出し例

### 調査プロジェクトの場合

```
1. skills/interview-research-project-workflow/SKILL.md を読み込む
2. ワークフローのステップ一覧を取得
3. 現在のステップに必要な連携スキルを確認
4. 連携スキルのSKILL.mdを読み込んで実行
```

## 新しいワークフロースキルの追加

新しいプロジェクト種類に対応するには:

1. `skills/` に新しいワークフロースキルを作成
2. SKILL.mdにフェーズ・ステップを定義
3. このスキルの「対応ワークフロー」表に追加
4. `references/status-template.md` に必要なフィールドを追加（あれば）
