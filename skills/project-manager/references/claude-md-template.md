# CLAUDE.md テンプレート

プロジェクト初期化時に、以下のテンプレートを使用して `CLAUDE.md` を作成する。
`{変数名}` をヒアリング結果で置換すること。動的セクション（論点サマリ・仮説ステータス・アプローチ・重要な意思決定）は初期化時は空のままにし、提案書作成後に自動で埋める。

---

```markdown
# プロジェクト: {プロジェクト名}

## 概要
- クライアント: {クライアント名}
- 種類: {project_type}
- 納期: {YYYY-MM-DD}
- スコープ: {スコープ}

## 論点サマリ
<!-- 提案書作成後に自動追加。変更があれば随時更新。詳細は Output/提案書.md 参照 -->

## 仮説ステータス
<!-- 提案書作成後に自動記載。検証の進行に応じて随時更新。詳細は Output/提案書.md 参照 -->

## アプローチ
<!-- 現在の調査方針・手法の要約。方針変更時に随時更新 -->

## 重要な意思決定
<!-- スコープ変更・方針転換・クライアント指示など、プロジェクトの方向性に影響する決定を記録 -->

## 作業上の注意
<!-- プロジェクト固有の前提条件・制約・表記ルールなど。3-5項目以内に留める -->

## ファイル配置
- Input/: クライアント提供資料・与件
- Input/インタビュー/: インタビュー文字起こし
- Output/: 全成果物
- Output/議事録/: インタビュー議事録
- Output/調査/: デスクリサーチ結果
- Meeting/: 会議の議事録・討議資料

## 利用可能なリソース
- スキル: skills/ 配下の全スキル
- SubAgent: agents/ 配下
- Agent Team Playbook: skills/agent-team-playbook/
- ワークフロー定義: workflow.md（カスタムワークフローの場合）
```

## 変数の説明

| 変数 | 取得元 | 例 |
|------|--------|-----|
| `{プロジェクト名}` | ヒアリング | AI動向調査プロジェクト |
| `{クライアント名}` | ヒアリング | 株式会社○○ |
| `{project_type}` | AskQuestion の選択結果 | technology-research / market-research / interview-focused / due-diligence |
| `{YYYY-MM-DD}` | ヒアリング | 2026-03-31 |
| `{スコープ}` | ヒアリング（1-2行） | 生成AI技術の市場動向と主要プレイヤーの戦略を調査し、参入機会を評価する |

## 動的セクションの更新ルール

CLAUDE.md の動的セクションは、project-manager の自動同期ルールに従い、ユーザーの明示指示なしに更新される。正本は `Output/提案書.md` であり、CLAUDE.md にはサマリのみを持つ。

| セクション | 内容 | 正本 | 更新契機 |
|-----------|------|------|---------|
| 論点サマリ | 大論点の問いを番号付きリストで列挙 | Output/提案書.md | 論点の追加・削除・変更時 |
| 仮説ステータス | 仮説文と状態1語（未検証/支持/修正/棄却）の表。根拠は書かず提案書参照 | Output/提案書.md | 仮説の追加・状態変化時 |
| アプローチ | 現在の調査方針・手法の要約（1-2行） | Output/提案書.md | 調査方針の変更時 |
| 重要な意思決定 | 日付と決定内容のリスト（追記型） | CLAUDE.md 自身が正本 | スコープ変更・方針転換時 |

### 更新後のイメージ

```markdown
## 論点サマリ
1. ○○市場の成長余地はどの程度か
2. △△技術の実用化時期はいつか
3. 参入障壁は何か

## 仮説ステータス
| 仮説 | 状態 |
|------|------|
| ○○市場は年率15%で成長する | 支持 |
| △△技術は2027年に実用化される | 修正 → 2028年 |
| 規制が最大の参入障壁である | 未検証 |

## アプローチ
デスクリサーチで市場構造を把握した上で、技術専門家・業界プレイヤーへのインタビューで仮説を検証する

## 重要な意思決定
- 2026-02-10: 初期調査の結果、○○領域はスコープ外とする（クライアント合意済み）
```

## 注意事項

- CLAUDE.md はプロジェクトルールとして全セッション（Cursor / Claude Code）で自動ロードされる
- 動的セクションは project-manager の自動同期ルールに従い、論点・仮説・アプローチ・タスクに変化が生じた時点で更新される。ユーザーへの確認は不要
- 論点・仮説・アプローチの詳細・根拠は Output/提案書.md（正本）を参照する。CLAUDE.md にはサマリだけを持ち、肥大化を防ぐ
- 仮説ステータスの状態カラムは1語（未検証/支持/修正/棄却）を基本とする。最小限の補足が必要な場合のみ10文字程度まで追記可（例: `修正 → 2028年`）。根拠・検証経緯は書かない
- 作業上の注意はプロジェクト固有の制約のみ記載する。一般的なルールは書かない。3-5項目以内に留める
- 重要な意思決定セクションは CLAUDE.md 自身が正本であり、追記型で運用する
- Agent Team 固有の運用ルールやタスクアサインは書かない（テンプレート側に記載する）
- スキル一覧は個別に列挙せず、ディレクトリを包括参照する
